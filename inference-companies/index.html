01 VALUE

推理行业的价值：把昂贵 GPU 变成更便宜、更稳定的 tokenThe Value of Inference: Turning Expensive GPUs Into Cheaper, More Reliable Tokens

推理公司不是在训练基础模型，而是在做一件更工程化的事：同样的模型、同样的 GPU，谁能每秒产出更多 token、首字延迟更低、服务更稳定，谁就能吃到优化差价。 Inference companies are not training foundation models. They are doing something more operational: given the same model and the same GPU, whoever produces more tokens per second, lowers latency, and runs more reliably captures the optimization spread.

成本Cost

GPU 是固定账单GPU spend is fixed

GPU 按小时烧钱。不管请求多不多、卡有没有跑满，账单都在走。推理优化的第一目标就是提高 GPU 利用率。GPUs burn money by the hour. Whether the card is full or idle, the bill runs. Inference optimization starts with keeping GPUs busy.

产出Output

token 是可优化产量Tokens are the output

更快的 kernel、更好的批处理、更少的重复计算，都会让同一块 GPU 每小时吐出更多 token，单位成本下降。Better kernels, batching, and cache reuse make the same GPU produce more tokens per hour, driving unit cost down.

体验UX

延迟决定产品体感Latency defines the product feel

首字等待和逐字速度直接决定 AI 应用是否“顺手”。推理公司把这些变成可承诺的 SLA。Time-to-first-token and streaming speed decide whether an AI product feels responsive. Inference companies turn them into SLAs.

主权Control

开源模型需要运行层Open models need a runtime layer

Llama、Qwen、DeepSeek 等开源模型给了客户选择权，但真正上线还需要推理、调度、监控和可靠性。Open models such as Llama, Qwen, and DeepSeek create choice, but production still needs serving, scheduling, monitoring, and reliability.

一句话：推理行业的价值，不是“又训练了一个模型”，而是把模型运行这件事做得更快、更便宜、更稳定，并把 GPU 成本优势转化为毛利和客户体验。 In one line: inference value is not "training another model." It is running models faster, cheaper, and more reliably, then turning GPU efficiency into margin and user experience.
02 TECHNOLOGY

推理主要做哪些技术：一张工具箱地图What Inference Companies Actually Build: The Toolbox

三家公司技术方向高度重叠。区别不在于“某项技术只有谁会”，而在于谁把哪一项做成招牌、做得多深、是否跑到生产级。 The technical directions overlap heavily. The difference is not that one company alone owns a technique; it is which technique each turns into a flagship, how deep the implementation goes, and whether it works in production.

技术Technology 大概是什么What it does 主要改善Main effect 谁最主推Who emphasizes it most
内核 / Attention 优化Kernels / attention optimization 重写 GPU 上真正执行计算的小程序，减少 HBM 与计算单元之间的数据搬运。Rewrite the low-level GPU programs that do the math, cutting data movement between HBM and compute. 更低延迟、更高吞吐，尤其是 Prefill 和注意力计算。Lower latency and higher throughput, especially in prefill and attention. Fireworks（FireAttention）和 Together（FlashAttention 科研）Fireworks via FireAttention; Together via FlashAttention research.
量化Quantization 把权重或 KV Cache 从 FP16 压到 FP8/FP4，减少数据体积。Compress weights or KV cache from FP16 to FP8/FP4, reducing bytes moved and stored. Decode 更快、显存装下更多并发。Faster decode and more concurrency in the same GPU memory. Fireworks 最常把它当招牌讲；三家都会用。Fireworks makes it a signature theme; all three use it.
KV Cache 管理KV-cache management 缓存前文算过的 Key/Value，做分页、压缩、复用、前缀缓存。Store prior Key/Value states, then page, compress, reuse, and prefix-cache them. 少重复计算，更多并发，更低首字等待。Less recomputation, more concurrency, lower time-to-first-token. Together 强调分页/压缩/复用；Baseten 强调 KV 感知路由。Together emphasizes paging/compression/reuse; Baseten emphasizes KV-aware routing.
连续批处理 / 调度Continuous batching / scheduling 把很多请求动态拼在一起跑，生成完的下车，新请求上车。Dynamically batch many requests: completed ones leave, new ones enter, and the GPU stays full. 提高 GPU 利用率，同时控制排队和 OOM。Higher utilization while controlling queues and OOM risk. Baseten 最强调生产调度；三家都必须做。Baseten emphasizes production scheduling; all three need it.
投机解码Speculative decoding 用小模型先猜，大模型一次性批改，猜对就省掉多步生成。A small model drafts ahead; the large model verifies in one pass, saving multiple decode steps when guesses are accepted. 加速逐字生成，通常用于 Decode。Speeds up token-by-token generation, mainly in decode. Fireworks 用 FireOptimizer 定制 drafter；Together 用 ATLAS 在线适应。Fireworks customizes drafters via FireOptimizer; Together adapts online with ATLAS.
P/D 分离Prefill/decode disaggregation 把 Prefill 和 Decode 分到不同 GPU 池，按两种工作负载分别扩容。Split prefill and decode into separate GPU pools that scale independently. 减少队头阻塞，让长输入和逐字输出互不拖累。Reduces head-of-line blocking so long inputs do not stall streaming output. Fireworks 公开作为架构卖点；大规模平台普遍会采用类似思路。Fireworks markets it openly; large-scale platforms generally use related ideas.
部署、冷启动、多云Deployment, cold starts, multi-cloud 把模型标准化上线，快速拉起 GPU，跨云调度容量和故障切换。Standardize model deployment, spin up GPUs quickly, and schedule/fail over across clouds. 可靠性、弹性扩容、客户少操心。Reliability, elastic scaling, and less operational burden for customers. Baseten 最主推；Together 在自有集群层也很重。Baseten leads this story; Together is also heavy at the owned-fleet layer.
03 COMPANIES

三家公司技术都做，但主推的产品形态不同They Share the Toolbox, but Sell Different Products

最重要的判断：技术栈趋同，商业包装和客户入口不同。 The key judgment: the technology stacks converge, but the commercial packaging and customer entry points differ.

Fireworks

卖更快更便宜的 tokenSells faster, cheaper tokens

  • 客户入口：“我不想自己跑 Llama/Qwen/DeepSeek，给我一个快 API。”Entry point: "I don't want to run Llama/Qwen/DeepSeek; give me a fast API."
  • 主推：FireAttention、FP8/FP4、P/D 分离、FireOptimizer。Emphasis: FireAttention, FP8/FP4, P/D disaggregation, FireOptimizer.
  • 商业形态：轻资产，租云上 GPU，靠软件优化吃差价。Business model: asset-light, rented cloud GPUs, monetizing software efficiency.
Together AI

卖 token + 算力Sells tokens plus compute

  • 客户入口：从 API 用量开始，也可以长成微调、训练、整租集群。Entry point: starts with API usage, can grow into fine-tuning, training, and cluster contracts.
  • 主推：FlashAttention 科研、FlashDecoding、ATLAS、GB200/Neocloud。Emphasis: FlashAttention research, FlashDecoding, ATLAS, GB200/neocloud.
  • 商业形态：重资产，自有或长租 GPU，软件优势叠加算力供给。Business model: capital-heavy, owned or long-leased GPUs, stacking software edge on compute supply.
Baseten

卖部署与可靠运行Sells deployment and reliable operation

  • 客户入口：“这是我自己的模型，帮我稳定上线。”Entry point: "Here is my own model; help me run it in production."
  • 主推：Truss、Engine Builder、连续批处理、KV 路由、多云、冷启动、SLA。Emphasis: Truss, Engine Builder, continuous batching, KV routing, multi-cloud, cold starts, SLAs.
  • 商业形态：轻资产平台，按 replica 运行时长和平台能力收费。Business model: asset-light platform, billing around replica runtime and managed operations.
更精确的说法：Fireworks 和 Together 更像“帮你用优化好的公共开源模型”；Baseten 更像“帮你的任意模型变成生产服务”。前两家卖推理结果，Baseten 卖生产运行能力。 More precise framing: Fireworks and Together are closer to "use our optimized public open models"; Baseten is closer to "turn your model into a production service." The first two sell inference output; Baseten sells production runtime.
04 MOATS

竞争壁垒：单点技术会被追，生产系统会沉淀Moats: Point Technologies Get Copied; Production Systems Compound

考虑 vLLM/SGLang 的影响，不能把任何单点技术都当永久壁垒。开源会把“及格线”不断抬高，真正能留住客户的是生产级系统、客户工作流和运营经验。 With vLLM/SGLang in the picture, no single technique should be treated as a permanent moat. Open source keeps raising the baseline. What retains customers is production-grade systems, customer workflow, and operating experience.

壁垒总判断：这三家最容易被高估的是“某个技术独家”；最容易被低估的是“生产系统长期调出来的参数、客户流程、故障经验和切换成本”。 Moat judgment: the easiest thing to overrate is "exclusive technology." The easiest thing to underrate is the compound value of tuned production parameters, customer workflows, failure experience, and switching cost.
05 RISKS

被上下游蚕食的可能性：谁会来抢中间层利润Erosion Risk: Who Can Attack the Middle Layer?

推理公司站在云、GPU、模型和应用之间。这个位置有价值，也天然会被上下游挤压。 Inference companies sit between clouds, GPUs, model labs, and applications. That middle position is valuable, but naturally exposed to pressure from both sides.

纯速度差异被追平Raw speed gets copied

开源和 NVIDIA 工具链会持续压缩单点技术溢价。只靠“我比 vLLM 快一点”很难长期防守。Open source and NVIDIA tooling keep compressing point-tech premiums. "We are a bit faster than vLLM" is hard to defend forever.

客户流程和生产责任Customer workflow and production ownership

一旦监控、扩缩、SLA、定制调优和故障经验长在平台上，迁移就不只是换 API，而是重建生产系统。Once monitoring, scaling, SLAs, custom tuning, and incident history live on the platform, switching is not just changing an API; it is rebuilding production.

最终判断：这不是一个“谁有独家技术”的行业，而是一个“谁能把趋同技术做成稳定产品、客户入口和运营复利”的行业。vLLM 会让基础推理越来越商品化，但也会把客户教育好，让他们更清楚什么时候该自建、什么时候该买专业平台。 Bottom line: this is not an "exclusive technology" industry. It is an industry about turning converging techniques into stable products, customer entry points, and operating compounding. vLLM commoditizes the baseline, but it also educates buyers: when to self-build, and when a specialist platform is worth paying for.
