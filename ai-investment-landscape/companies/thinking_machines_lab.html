Thinking Machines 的差异化在于:它不是单纯发布一个闭源聊天模型,而是把 frontier lab 内部用于 post-training、LoRA、RL、distributed training 的能力封装成 API,让外部研究者和企业在不用自己管理 GPU 集群的情况下训练/微调开放模型。它的核心客户可能不是普通消费者,而是研究型开发者、AI startup、大学实验室、企业 AI 团队和需要定制模型的行业客户。

4业务模式与收费方式

4.1 当前产品:Tinker training API

Tinker 是 Thinking Machines 第一款公开产品。官网把它定义为“training API for researchers”:用户控制数据、算法、训练循环和保存 checkpoint,Thinking Machines 负责分布式训练基础设施、调度、资源管理和可靠性。核心 API 抽象只有四类函数:

函数作用商业含义
forward_backward前向和反向传播,累积梯度直接对应训练 compute 消耗
optim_step根据梯度更新权重使用户可控制训练算法
sample生成 tokens,用于交互、评估或 RL action可按采样 tokens 收费
save_state保存训练进度,便于恢复形成 storage 和 checkpoint 收费

与传统 API 相比,Tinker 卖的不是一个固定模型输出,而是可管理的训练过程。这使它更接近“frontier training infrastructure as a service”。

5产品与技术

5.1 公司技术路线

Thinking Machines 官网把使命写成三件事:让 AI 更可理解、更可定制、更通用。技术路线包括:

  • human-AI collaboration:不只追求完全 autonomous agents,而是构建能与人协作的 multimodal systems;
  • customizable AI:让模型适配具体领域、工作流和价值偏好;
  • frontier model intelligence:在科学、编程等领域推进前沿模型能力;
  • advanced multimodality:语言、视觉、音频等更自然的交互方式;
  • infrastructure quality:强调研究生产力、可靠性、安全和长期正确性。

5.2 Tinker 支持的模型生态

官网显示 Tinker 支持多类 open-source/open-weight 模型,包括:

  • Qwen:Qwen3.6-35B-A3B、Qwen3.6-27B、Qwen3.5-4B/9B/35B-A3B/397B-A17B 等;
  • GPT-OSS:GPT-OSS-120B、GPT-OSS-20B;
  • DeepSeek:DeepSeek-V3.1;
  • Moonshot/Kimi:Kimi-K2.5、Kimi-K2.6;
  • NVIDIA Nemotron:Nemotron-3-Nano/Super/Ultra。

这说明 Tinker 的定位不是替代开源模型,而是成为 open-weight model training layer。它让用户选择基础模型,再用 LoRA、supervised learning 或 reinforcement learning 做定制,最后下载 checkpoint 或部署。

5.3 LoRA 与 post-training

Tinker 使用 LoRA 微调,即训练一个小型 adapter 而不是改动全部 base weights。John Schulman 及团队在 2025 年发布 “LoRA Without Regret” 研究,强调在合适设置下 LoRA 可以接近 full fine-tuning 的学习表现,同时降低计算和部署复杂度。对商业模式而言,LoRA 让 Tinker 能服务更多中小团队:用户不需要全量训练 100B+ 模型,但仍可获得领域定制能力。

5.4 Research output

2025-2026 年公司已经发布多篇研究/技术博客:

  • “Defeating Nondeterminism in LLM Inference”:关注推理确定性和可复现性;
  • “LoRA Without Regret”:关注高效 fine-tuning;
  • “On-Policy Distillation”:关注策略蒸馏;
  • “Interaction Models”:关注人机协作的可扩展交互模型;
  • “Learning to Replicate Expert Judgment in Financial Tasks”:用金融任务专家判断作为研究对象;
  • “Training LLMs to Predict World Events”:与 Mantic 的 guest post,关注预测类任务。

这些研究主题与 Tinker 产品高度相关:训练效率、可复现性、RL/post-training、人机交互和专家任务评估,都是把前沿研究能力转化为可用训练平台的基础。

5.5 NVIDIA gigawatt-scale partnership

2026-03-10,Thinking Machines 和 NVIDIA 宣布长期 gigawatt-scale strategic partnership。公开资料提到未公开金额投资、多年协议,以及部署 1 gigawatt Vera Rubin computing capacity。对模型公司来说,这相当于把未来算力供给前置锁定,其重要性接近甚至超过单纯股权融资:

  • 为 Tinker 和自研 frontier systems 提供训练/推理基础;
  • 让 Thinking Machines 成为 NVIDIA 新一代硬件生态的旗舰客户之一;
  • 给外部客户传递“可持续供给”的信号;
  • 也增加对 NVIDIA 硬件周期、功耗、数据中心建设和资本开支的依赖。

7单位经济性 / 关键财务逻辑

Thinking Machines 的财务逻辑比 SSI 更可建模,因为 Tinker 已经收费。

7.1 Tinker 单位经济性

Tinker 收入大致来自:

收入 = prefill tokens × 单价 + sample tokens × 单价 + train tokens × 单价 + storage GB-month × $0.10 + enterprise SLA / reserved capacity

成本大致来自:

成本 = GPU/加速器折旧或租赁 + 电力 + 网络 + 存储 + 调度/平台工程 + support + 安全审核

关键变量:

  • GPU utilization:训练 API 如果 job 碎片化,利用率低会伤毛利;
  • 模型 mix:Nemotron Ultra / Qwen large MoE / Kimi / DeepSeek 等大模型成本差异很大;
  • 上下文长度:256K context 价格显著更高,同时 KV cache 和 memory bandwidth 成本更高;
  • LoRA adapter:比 full fine-tune 成本低,有助于服务长尾用户;
  • NVIDIA partnership:若能锁定低成本或优先 capacity,毛利空间更好;
  • enterprise reserved capacity:比纯按量 self-serve 更稳定,也可降低空置风险。

7.2 与普通 inference API 的区别

普通 inference API 的成本主要是 serving tokens;Tinker 的成本包含训练、采样和保存 checkpoint,更接近 cloud compute。优点是付费意愿更高:客户微调模型是为了获得差异化能力,不是只买 commodity tokens。缺点是需求更窄:普通开发者会直接调用 OpenAI/Anthropic/DeepSeek API,只有需要定制模型和 RL 的团队才会用 Tinker。

7.3 估值逻辑

以 $12B 估值看,Thinking Machines 需要证明至少一种大路径:

  • 成为 open-weight model post-training 的默认平台,类似“frontier AI training cloud”;
  • 发布自有 frontier model,进入 OpenAI/Anthropic/xAI 级别竞争;
  • 用 NVIDIA 合作获得显著成本/供给优势,成为企业训练和 agent workload 的基础设施;
  • 通过 Tinker 建立开发者生态,再向企业平台和多模态协作产品扩展。

8.3 护城河

Thinking Machines 的护城河可能来自四层:

  1. 团队:Mira Murati、John Schulman 等前 OpenAI 核心人物带来招聘和融资优势。
  2. 训练 API know-how:把 RL/post-training/distributed systems 封装成稳定 API,比裸 GPU 更难复制。
  3. 算力供给:NVIDIA 1GW Vera Rubin 合作若落实,是非常强的基础设施壁垒。
  4. 开发者生态:Tinker 若成为研究者微调 open models 的默认工具,会积累 cookbook、adapter、eval、社区项目和数据流。

9团队与背景

Mira Murati

Mira Murati 曾任 OpenAI CTO,参与 DALL-E、ChatGPT、GPT-4 时代产品和研究工程管理,并在 2023 年 OpenAI 董事会危机期间短暂担任 interim CEO。她的优势是兼具技术、产品和组织经验,比纯研究型 founder 更适合把 frontier lab 能力产品化。

John Schulman

John Schulman 是 OpenAI 联合创始人,长期研究 reinforcement learning、policy optimization、RLHF,是 ChatGPT post-training 路线的重要人物。Tinker 把 RL/fine-tuning 开放成 API,与 Schulman 的技术背景高度一致。

其他核心成员

  • Barret Zoph:前 OpenAI VP of Research/Post-Training,2026 年被报道离开回到 OpenAI;
  • Lilian Weng:前 OpenAI 安全/机器人/研究负责人之一;
  • Andrew Tulloch:前 OpenAI/Meta 背景,2025 年被 Meta 挖走;
  • Luke Metz:post-training specialist,2026 年被报道离开回到 OpenAI;
  • Alec Radford、Bob McGrew 等被报道为 adviser 或早期相关人物。

团队风险在于:Thinking Machines 的最大资产也是最容易被攻击的资产。Meta/OpenAI/Anthropic/Google 能用现金、股权、算力和研究资源直接挖人。2025-2026 年几次核心人员流动说明公司需要从“明星名单”转向“可持续组织能力”。

10主要风险

  1. 估值先于收入:$12B 估值建立在团队、产品期权和算力合作上,而非公开 ARR。
  2. 人才流失:Meta 和 OpenAI 已经挖走/吸回部分关键成员。
  3. 产品需求深度:Tinker 面向高端研究者和开发者,市场规模可能小于通用 chat/API。
  4. 安全与滥用:开放 fine-tuning 和 RL 能力可能被用于生成危险能力或绕过安全限制,需要强 vetting。
  5. 数据信任:企业是否愿意把私有训练数据传到 Tinker,取决于安全、隔离和合规。
  6. 算力资本开支:1GW 级合作意味着巨大的电力、数据中心和硬件周期风险。
  7. open-weight 依赖:Tinker 价值部分依赖 Qwen/DeepSeek/Kimi/Nemotron 等外部模型生态。
  8. 大厂平台夹击:OpenAI、Google、Anthropic、AWS、Microsoft、NVIDIA 自己都可向 training platform 延伸。
  9. 监管:训练 API 让用户生成自己的模型,责任边界和模型输出责任更复杂。
  10. 治理集中:创始人控制权保护长期主义,也可能降低投资人对风险控制的影响。
