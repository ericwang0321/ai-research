Bioptimus

Bioptimus 是 AI biology 中商业化路径较清晰的一类公司。它不是从单个药物管线切入，而是用 H-Optimus pathology foundation model、M-Optimus multimodal biology model 和 STELA 临床多模态数据引擎服务药企、研究机构和临床转化团队。

Bioptimus 的差异化不是只做病理图像模型。H-Optimus 先以 H&E whole-slide images 建立 pathology foundation model 标准,M-Optimus 则试图把 tissue、molecular、clinical data 放到统一 embedding space 中,STELA 负责生成和汇聚多机构、临床 linked、spatial biology/molecular/pathology 数据。这是典型的“模型 + 数据引擎 + 药企转化合作”三位一体结构。

4业务模式与收费方式

4.1 H-Optimus license 与部署

H-Optimus 的商业化路径较清楚:

  • Academic license:H-Optimus-0 以 Apache 2.0 开源;H-Optimus-1 和 H0-mini 面向学术研究为 CC-BY-NC-ND 4.0,商业用途需要单独 license;
  • AWS SageMaker / Marketplace:H-Optimus-1 可通过 AWS 部署,客户在自有安全云环境中 evaluate、fine-tune、deploy;
  • Industry license:面向 pharma/clinical partners,用于 biomarker development、patient stratification、translational research;
  • Collaboration:与药企或 cancer center 共同开发下游模型、biomarker signatures、trial enrichment 方法。

这意味着 Bioptimus 不一定靠公开 API token 收费,更可能通过年度 license、secure deployment、project collaboration、数据/模型共建和里程碑式成功费用变现。

4.2 M-Optimus 商业化

M-Optimus 是更高层的 multimodal/multiscale model,目标是从 H&E、routine sequencing、spatial transcriptomics 和 clinical outcomes 中学习 unified patient representation。官网强调它可以:

  • predict gene expression;
  • predict patient stratification and outcomes;
  • predict drug targets;
  • support in-silico trial design;
  • identify biomarkers earlier;
  • prioritize indications and combinations。

4.3 STELA 数据引擎与合作模式

STELA 是 Bioptimus 的长期护城河。官网将其描述为跨机构、跨疾病、跨地理的 multimodal patient resource,从 tissue architecture、spatial biology、molecular assays、clinical outcomes 中生成数据,用于训练 M-Optimus。合作机构贡献患者样本、组织、临床记录或既有数据;Bioptimus 提供 spatial omics profiling、M-Optimus models 和论文/科研合作。

STELA 的经济逻辑不是简单买数据,而是“共同生成数据 → 模型提升 → 合作方获得分析能力 → 药企/临床开发获得更高质量 biomarker/trial insight”。如果能实现,它会形成数据网络效应。

4.4 不能直接写药品收入

Bioptimus 不是传统 biotech pipeline 公司。公开资料没有显示它拥有临床药物资产、IND、临床试验或药品 royalty。它更像平台公司,收入来自 software/license/collaboration。写作上应避免把 “drug discovery & translational research” 误写成“已有药品销售”。

5.1 H-Optimus

H-Optimus 是 Bioptimus 的病理 foundation model family。

模型参数/训练数据License/入口说明

H-Optimus 不直接给出诊断结论,而是作为 embedding backbone:把 whole-slide image 切成 224x224 tiles,经 ViT encoder 形成 embeddings,下游再训练 MIL heads 或 specialized heads 做 biomarker prediction、grading/prognosis、gene expression estimation、tissue/cell segmentation、treatment response 等任务。

5.2 H-Optimus benchmark

官网称 H-Optimus-1 在 PathBench 229 tasks 上总体排名第一,并列出 H-Optimus-1、Virchow2、H-Optimus-0、UNI2、mSTAR 等比较。官网和独立文献还提到:

  • H-Optimus-1 在 biomarker prediction、spatial gene expression、survival prediction 等下游任务表现强;
  • 2026 年 breast cancer survival benchmark 提到 H-Optimus-1 在多 cohort survival prediction 中表现最佳;
  • Phikon-v2 论文将 H-Optimus-0 与 GigaPath 等列为高性能 pathology foundation models。

投资解读:benchmark 是 adoption 的基础,但 clinical deployment 仍需要外部验证、local scanner/stain validation、prospective study 和 regulatory pathway。

5.3 M-Optimus

M-Optimus 是 Bioptimus 从 pathology 扩展到 multimodal biology 的关键。它不是只看 H&E slides,而是整合:

  • histology images;
  • spatial transcriptomics;
  • genomics / sequencing;
  • clinical records;
  • clinical outcomes。

官网称其可预测 gene expression、patient stratification/outcomes、drug targets,并用于 in-silico trial arms。商业价值在于把已有 archival H&E slides 和临床记录变成可挖掘的生物学信号,帮助药企在早期就识别患者子群和适应症扩展机会。

5.4 STELA

STELA 全称 Spatial Tissue Embedding Learning Atlas,目标是构建最大规模 clinical linked spatial biology atlas。2026-03 公告称 STELA 计划 profile up to 100,000 patient specimens,规模约为当时世界可用数据的 20x increase。合作方包括:

  • 10x Genomics:founding spatial transcriptomics platform partner,使用 Xenium in situ 技术;
  • Broad Clinical Labs:single-cell 和 spatial multi-omics 数据生成能力;
  • leading cancer centers:美国、欧洲、亚洲 prospective cohorts,部分名称需 NDA;
  • FFCD:2026-06 合作,接入 3,000+ gastrointestinal cancer patients clinical trial data。

5.5 实验验证与临床验证边界

Bioptimus 目前的验证更多是 retrospective cohort、benchmark、case study 和研究合作,而不是监管批准诊断。官网 FAQ 明确提示 H-Optimus 适合 clinical development/translational exploratory research,但不是已获 FDA/EMA clearance 的 regulated diagnostic 或 companion diagnostic。用于病人层面决策时仍需要 external validation、pre-specified endpoints、prospective bridge studies 和 confirmatory molecular assays。

这点非常重要:商业化可以发生在药企研发和临床试验设计中,但不能把它写成已可直接替代病理医生或伴随诊断。

7单位经济性 / 关键财务逻辑

7.1 License / deployment economics

H-Optimus 和 M-Optimus 的商业毛利取决于部署方式:

  • AWS/SageMaker:客户承担云资源,Bioptimus 收 license/usage/revenue share,毛利可能较高;
  • private enterprise deployment:收入较高,但需要 support、security、validation、integration;
  • academic license:收入低或免费,但提升引用和 adoption;
  • commercial license:取决于用途、机构规模和是否包括下游模型开发。

7.2 Pharma project economics

药企合作可能按以下结构:

project fee + data processing fee + model license + biomarker/trial milestone + optional success fee

关键不是“每张 slide 收多少钱”,而是能否帮助药企:

  • 更早发现 biomarker;
  • 提高 trial enrichment;
  • 降低 Phase II/III 失败率;
  • 找到 indication expansion;
  • 用 archival slides 生成新 biological readouts。

如果 M-Optimus 能提高临床开发成功概率,客户愿意支付的金额会远高于普通 pathology AI software。

7.3 STELA 数据网络经济性

STELA 的成本包括:

  • specimen profiling;
  • spatial omics / genomics / imaging 数据生成;
  • 数据标准化和合规;
  • 多机构协议和伦理审批;
  • 模型训练与验证。

收益来自:

  • 数据越多,模型越强;
  • 合作机构获得模型和分析能力;
  • 药企获得更强 biomarkers/trial design;
  • Bioptimus 获得差异化 proprietary/collaborative data moat。

7.4 推理成本

H-Optimus-1 是 1.1B parameter ViT-g/14 pathology encoder,处理 whole-slide images 需要大量 tile extraction 和 embedding。H0-mini 的战略价值是降低 feature extraction 成本,让更多临床/研究场景可部署。未来商业化中,模型大小、slide volume、batch processing 和云部署效率会直接影响 gross margin。

9团队与背景

Bioptimus 团队强调 ML 和 biology 双能力。官网称团队来自 Google Brain、Google DeepMind、Meta、Owkin、MIT、ETH Zürich、Cambridge、Helmholtz 等,60% 团队成员拥有 PhD。公司还在 2025 年强化商业和科学 leadership:

  • 2025-07 任命 Julie Gerardi 为 Chief Commercial Officer,强化 biotech commercialization 和 pharma partnership;
  • 2025-08 增加 Ketan Patel 与 Tokuwa Kanno 等 commercial/scientific leadership;
  • 2025-09 组建 scientific advisory board,八位国际专家支持 biology foundation model 方向;
  • Mathilda Strom 于 2024 年加入为 founding COO,负责组织扩张。

团队结构上,Bioptimus 相比纯模型公司更接近 pharma-facing AI company:除了 ML 研究,还需要 clinical operations、pathology、spatial biology、business development、regulatory validation 和 cloud deployment。

10主要风险

  1. 官网 adoption 指标与收入之间的差距:downloads、papers、institutions 不等于付费合同。
  2. 临床验证风险:retrospective benchmark 强,但 prospective clinical utility 需要长期验证。
  3. 监管风险:H-Optimus 不是已获批诊断或 CDx,若用于病人决策需要额外验证。
  4. 数据合规风险:STELA 跨国家/机构/临床数据,涉及隐私、同意、伦理审批和数据权属。
  5. 模型泛化风险:scanner、staining、population、disease mix 变化可能影响表现。
  6. 竞争风险:Virchow、GigaPath、UNI、Phikon、PathAI、Paige 等均在 pathology AI 领域强势。
  7. 商业周期长:药企和临床研究网络采购、验证、合规周期较长。
  8. 成本风险:spatial omics 和 multimodal profiling 成本高,STELA 需要持续资本。
  9. IP / license 复杂性:academic license、commercial license、AWS deployment、partner data 权利需要清晰。
  10. 临床责任边界:模型作为 research/development backbone 与实际诊断责任必须清楚区分。
