Baseten

Baseten 是 AI 应用从闭源 API 走向开放模型、自定义模型和多云推理部署时的生产运行层。它的核心产品不是传统 MLOps dashboard，而是把 open-source/custom models 打包成可自动扩缩容、可观测、低延迟、按分钟或按 token 计费的推理平台。

1一句话结论

Baseten 是 AI 应用从闭源 API 走向开放模型、自定义模型和多云推理部署时的生产运行层。它的核心产品不是传统 MLOps dashboard，而是把 open-source/custom models 打包成可自动扩缩容、可观测、低延迟、按分钟或按 token 计费的推理平台。

2026 年 6 月，公司宣布 $1.5B Series F，估值 $13B。官方称过去一年 revenue 增长 20x、inference volume 增长 40x，客户包括 Cursor、Notion、Lovable、Harvey、HubSpot、OpenEvidence、Abridge、Decagon、Parallel 等。这说明 Baseten 已经进入 AI-native 应用的核心生产链路。

Baseten 的定位和 Fireworks/Together 相近,但产品叙事更强调“embedded engineering”和“own your intelligence”。它不是简单提供模型列表,而是让客户把专有数据、evals、domain harness、post-training、custom RL environments 和 inference serving 连成一个持续优化系统。对于 Harvey、Abridge、OpenEvidence、Cursor 这类 AI-native 应用,模型不是后台工具,而是产品体验本身;Baseten 因此进入客户核心生产链路。

4业务模式与收费方式

4.1 Basic / Pro / Enterprise: 平台层套餐

Baseten 官方 pricing 将账户/平台层分为 Basic、Pro、Enterprise:

套餐公开口径核心权益投资解读
Basic$0/month,pay as you godedicated deployments、Model APIs、Training、fast cold starts、SOC 2 Type II、HIPAA、email/in-app support自助开发者入口,降低试用阻力
Proquote / volume discountpriority access to high-demand GPUs、dedicated compute、higher Model API rate limits、hands-on engineering、Slack/Zoom support从自助走向生产客户
Enterprisequote / volume discountcustom SLAs、self-host deployments、on-demand flex compute、use existing cloud commitments、data residency、advanced security/RBAC、custom regions面向大客户和 regulated deployments

这一结构说明 Baseten 的真正收入不在 seat fee,而在 compute、token、training 和 enterprise commitments。Basic 用于拉开发者;Pro/Enterprise 才是 ACV 和毛利空间。

4.2 Model APIs: 预优化模型按 token 收费

Baseten Model APIs 提供 pre-optimized models,按 1M tokens 计价。公开价格页列出的例子包括 GLM 5.2 input $1.40/M、cache input $0.26/M、output $4.40/M;Kimi K2.7 Code input $0.95/M、cache $0.16/M、output $4.00/M;DeepSeek V4 input $1.74/M、cache $0.145/M、output $3.48/M;GPT OSS 120B input $0.10/M、output $0.50/M。Model API 与 Fireworks/Together 的 serverless API 类似,但 Baseten 同时强调可以进一步迁移到 dedicated deployment 和自定义模型。

4.3 Dedicated Deployments: 按分钟/小时计费的生产推理

Baseten 的 Dedicated Deployments 只为实际使用 compute 付费,按分钟/小时计价。公开价格包括 T4 $0.01052/min、L4 $0.01414/min、A10G $0.02012/min、A100 $0.06667/min、H100 MIG $0.0625/min、H100 $0.10833/min、B200 $0.16633/min。换算小时,H100 约 $6.50/hour,B200 约 $9.98/hour。

Dedicated 适合客户部署:

  • 自己 fine-tuned 的模型;
  • 从 Hugging Face/开源社区拿到的模型;
  • 特殊依赖或自定义 runtime;
  • 需要 autoscaling、cold starts、observability 和 SLA 的生产 API。

Baseten 的卖点是客户不为 idle time 付费,只为模型部署、扩缩容和预测时使用的 compute 付费。这能降低企业从 prototype 到 production 的心理门槛。

4.4 Training 与 post-training

价格页显示 Training 使用同一组 GPU/CPU instance 计价,面向 fine-tuning、post-training、custom training jobs。2026 Series F 公告中,Baseten 特别强调客户正在构建由 proprietary data、evals、domain-specific harnesses、post-training 和 custom RL environments 形成的学习系统。这说明 Baseten 已经从“deploy a model”扩展到“持续改进模型”。

Training/post-training 的收入价值高于一次性 deployment,因为它连接客户私有数据和工作流。客户模型越依赖 Baseten 的 eval、数据、训练和 serving 闭环,迁移成本越高。

4.5 Enterprise contract: 自有云承诺、数据驻留和工程嵌入

Baseten Enterprise 支持 self-host、hybrid、customer VPC、custom global regions、data residency、advanced RBAC 和 custom SLAs。它还突出“hands-on engineering expertise”“forward deployed engineers”。这意味着大客户合同可能包含:

  • 年度 committed spend;
  • GPU capacity reservation;
  • 专属工程团队或 solution architect;
  • 安全、合规、HIPAA/SOC 2、区域部署;
  • 与客户现有 AWS/GCP/Azure commitments 对接;
  • custom model runtime、observability、eval pipeline。

这类合同更像 Palantir/Databricks 式的 enterprise platform + infrastructure spend,而不是纯 API 自助消费。

7单位经济性 / 关键财务逻辑

Baseten 的财务模型核心是“客户节省的闭源 API 成本,有多少能被 Baseten 捕获”。例如客户原来使用 OpenAI/Anthropic 完成某个任务,成本为 100;如果开放模型 + Baseten 运行成本为 30,理论节省 70。Baseten 可以用平台费、GPU margin、engineering service、enterprise commitment 捕获其中一部分,客户保留剩余节省。

关键变量:

  1. GPU cost: Baseten 从 20+ cloud providers sourcing compute,说明它通过多云采购、区域套利和供应商竞争降低成本。
  2. Utilization: autoscaling 和按分钟计费对客户友好,但平台端必须控制 idle 和 cold-start 成本。
  3. Software efficiency: runtime 优化、batching、quantization 和 model-specific tuning 决定每 GPU tokens/sec。
  4. Support intensity: embedded engineering 能拿大客户,但人力成本高;需要看 support cost 占收入比例。
  5. Revenue mix: Model API 更像 usage SaaS;dedicated deployment 更像 compute margin;enterprise/self-host 更像软件+服务合同。

$13B 估值下,即便按高增长 infra software 看,公司也需要很高 ARR 或明确可见的收入 run-rate。若 revenue 20x 是从较小基数增长,估值需要未来增长兑现;若已有数亿美元 ARR 且 gross margin 健康,则可支撑更高倍数。公开材料没有给出具体 ARR,这是最重要的数据缺口。

  • 高质量 AI-native 客户带来的生产反馈;
  • 多云 GPU sourcing 与调度经验;

  • 开源 runtime 进步会降低平台差异;

  • 模型 API 价格战会压缩 token margin;
  • $13B 估值提高执行容错要求。

9团队与背景

Baseten 由 Tuhin Srivastava、Amir Haghighat、Phil Howes、Pankaj Gupta 等创立。团队早期做的是让机器学习应用更容易构建和部署,后来随着生成式 AI 需求爆发,产品重心转向 inference platform。这个转型很重要: 传统 ML app platform 市场天花板有限,而 LLM inference 是所有 AI 应用的生产成本中心。

2026 Series F 公告由四位创始/高管共同署名,强调公司与客户“closely work”和“researchers and engineers embed with customers”。这说明团队文化偏产品工程和客户深度合作,而不是纯自助 SaaS。对于 Harvey/Abridge/OpenEvidence 这类高合规行业客户,这种工程支持可能是赢单关键。

10主要风险

  1. 估值风险: $13B 对尚未公开 ARR 的公司要求极高。
  2. 毛利风险: GPU 成本、云供应商价格和客户折扣会吞噬平台收益。
  3. 客户集中风险: AI-native 大客户贡献高,但任何一个自建/换供应商都会影响增长。
  4. 价格战风险: Fireworks、Together、云厂商和模型公司持续压价。
  5. 自建风险: 大客户可用 vLLM/TensorRT-LLM/SGLang 构建内部 serving。
  6. 服务成本风险: forward deployed engineering 有助于成交,但可能拉低 operating leverage。
  7. 模型风险: 开放模型若质量停滞,替代闭源 API 的市场空间受限。
  8. 合规风险: 医疗、法律、企业数据部署必须满足严格安全和审计要求。
