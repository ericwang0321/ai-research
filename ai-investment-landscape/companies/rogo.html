Rogo

Rogo 是面向投行、PE、资管和金融专业服务团队的 AI 研究与交付平台。它把 market data、filings、research、SharePoint/CRM、Excel/PPT、公司内部资料和金融专用模型连接起来，用于生成可审计的模型、memo、diligence materials 和 slide decks。

1一句话结论

Rogo 是面向投行、PE、资管和金融专业服务团队的 AI 研究与交付平台。它把 market data、filings、research、SharePoint/CRM、Excel/PPT、公司内部资料和金融专用模型连接起来，用于生成可审计的模型、memo、diligence materials 和 slide decks。

官网披露 35,000+ bankers and investors using Rogo、50,000+ daily queries、250+ institutions served。2026 年 4 月，公司宣布 $160M Series D，由 Kleiner Perkins 领投。投资重点是 Rogo 能否在高 ACV、强安全、强定制的金融机构里跑出可规模化毛利，并抵御 AlphaSense/Hebbia、Bloomberg/FactSet/S&P、Microsoft/OpenAI 和大行自建平台。

Rogo 的产品语言非常金融化。它不把自己包装成普通 enterprise search,而是说“Just as Bloomberg digitized financial data in the 1980s, Rogo is now transforming financial workflows.” 这句话定义了它的战略野心:Bloomberg/FactSet/S&P 已经把数据数字化,下一层价值是让 banker/investor 把数据、文档、模型、deck、memo 和客户沟通工作自动化。

4业务模式与收费方式

4.1 收费方式:bespoke enterprise deployment,没有公开 self-serve 价格

Rogo 官网入口是 Request Demo / Request Access,没有标准 pricing table。首页明确写“Every firm is different. That’s why every Rogo deployment is bespoke.” 产品页还写 guided implementation 由 ex-bankers and private equity investors 白手套参与。可确认的商业模式是金融机构 enterprise contract,可能按 seat、团队/部门、数据连接、workflow 模块、部署方式和用量打包。

4.2 价格锚:用 AI 替代 analyst/associate 的重复执行,释放 senior banker 时间

Rogo 的 ROI 主要来自:

  • junior banker 少花时间 shell deck、refresh materials、turn comments;
  • analyst/associate 更快 build model、spread comps、update backup;
  • senior bankers 更快做 client meeting prep、company profile、personal bio、sponsor overview;
  • 投资团队更快做 diligence、IC memo、market scan、portfolio monitoring;
  • 输出保持 auditable、consistent、firm-style,减少返工和风险。

Truist、Nomura、Baird 的官网 quote 都指向相同价值:boost productivity、reduce risk、create capacity、analyze market data faster、improve consistency in deliverables、senior bankers more autonomously get information、junior banker work-life balance 和 talent retention。

4.3 数据授权与合同边界

Rogo 集成大量金融数据源,但公开页面没有说明哪些数据由 Rogo 转售,哪些由客户自带 entitlement。金融数据许可很关键:

  • Bloomberg/FactSet/S&P/LSEG 等数据通常有严格使用和再分发限制;
  • 若客户已有合同,Rogo 可能作为 authenticated access / workflow layer;
  • 若 Rogo 自带数据包,COGS 和授权谈判会更重;
  • AI agent 用于生成 deck/model 可能涉及版权、research redistribution 和 compliance review。

投资尽调需要拆合同条款:数据成本由谁承担、是否按数据源加价、是否可跨团队共享、输出是否有来源和 redistribution 限制。

5产品与技术

5.1 平台核心能力

官网 product 页面把 Rogo 拆为四个能力:

  1. All your content in one place: 把公司内部文件、数据源、market data、research、filings、proprietary docs 统一。
  2. Transparent, auditable sources: 生成结果需要可追溯,适合投行/投资/合规。
  3. Automate your workflows: earnings comp analysis、meeting prep、company profile、proofread deck 等 workflow。
  4. Proprietary document interrogation: 对私有文档、数据室、内部材料做问答、抽取、比较。

这比 generic chatbot 更接近金融团队的实际流程。金融工作不是“问一个问题”,而是“从多个数据源抽证据,更新 Excel,生成 PowerPoint,写 memo,给 MD/IC/客户能看的东西”。

5.2 Felix:个人化金融 agent

Felix 页面强调“AI in finance has been treated like a better search box”,而 Felix 改变的是执行层。用户既可以在界面探索 ideas,也可以像给同事发 email 一样把更复杂工作委派给 Felix,并随着交付物推进持续迭代。Felix 覆盖:

  • Banking;
  • Private Markets;
  • Public Markets;
  • Decks: shell the deck, refresh materials, turn comments;
  • Spreadsheets: build the model, spread the comps, update the backup;
  • Reports: draft the memo, prep the meeting, run the research。

这正是投行和买方工作中耗时最多、但又要求格式/口径/数据一致性的部分。Felix 如果能做到“90% of the way there”并让人类做最后判断,就能提供强 ROI。

5.3 数据与系统集成

Rogo 产品页列出 trusted data providers 和数据类型:

  • LSEG;
  • Dow Jones;
  • FactSet;
  • Capital IQ;
  • PitchBook;
  • Preqin;
  • Real-time Web & News;
  • SEC Filings;
  • Transcripts;
  • Quartr;
  • International Filings;
  • Your Firm’s Data;
  • SharePoint、CRM、file systems。

这使 Rogo 的位置更像“finance workflow orchestration layer”。它不必拥有所有数据,但必须保证 entitlement、permission、citation、freshness 和 audit trail。金融客户尤其关心 output 是否能追溯到 filing、research note、transcript、database field 或内部文件。

5.5 安全、治理与部署

Rogo 产品页强调:

  • no training on your data;
  • modern secure data practices;
  • end-to-end encryption;
  • audited & tested;
  • granular permission controls;
  • role-based access management;
  • comprehensive audit trails;
  • customizable governance policies;
  • single tenant deployment options。

对大行/资管/PE 来说,这些不是可选项。没有 RBAC、audit trail、single tenant、data isolation,AI 工具无法进入真实 deal flow。Rogo 的白手套部署和 ex-finance team 是销售优势,但也带来交付成本。

6.1 用户与查询量解读

35,000+ bankers and investors 与 50,000+ daily queries 表明 Rogo 已经进入真实使用。粗略看,平均每用户每日 query 约 1.4 次,但这个数字可能被 inactive seats、team seats 或 heavy users 扭曲。金融 workflow 的关键不只是 query 数,而是 query 是否对应高价值 deliverable:

  • 一次 “build buyer universe” 可能比几十次普通问答更有价值;
  • 一次 deck/model 自动化节省的时间可能以小时计;
  • senior banker 的 meeting prep 价值高于 junior 的一般搜索;
  • 如果 Rogo 产生的材料能直接进入 client deck,其价值远高于 research assistant。

6.2 客户质量

官网 quote 来自 Truist Securities CEO、Nomura International Head of Investment Banking、Baird Global Investment Banking COO,这些是高质量背书。金融客户采用外部 AI agent 的门槛很高,尤其在投行涉及 MNPI、client confidential info、compliance 和 data licensing。能进入这些机构说明 Rogo 在安全和工作流落地上有一定证明。

6.3 收入透明度

公开资料没有 ARR。FT 报道 2025 年 $50M Series B at $350M valuation;2026 年 $160M Series D 未在官方公告中披露估值。用户数和机构数可以说明规模,但不能替代 ARR。后续应核验:

  • 35,000+ users 中 paid active users 比例;
  • 250+ institutions 的付费 ACV 分布;
  • query / workflow / document processing 是否收费;
  • expansion from pilot to firmwide deployment 的周期;
  • Series D 后估值和收入 run-rate。

7单位经济性 / 关键财务逻辑

7.1 Revenue build

Rogo revenue 可拆为:

ARR = 机构数 × 平均部门数 × paid users × 每 user / platform fee × workflow / data / deployment premium × usage/overage

关键变量:

  • Institution expansion: 从投行一个 coverage team 扩到全 IB、equity research、capital markets、wealth/asset management。
  • Seat density: banker/investor seat 单价高,但用户群比 horizontal SaaS 小。
  • Workflow attach: Felix、deck/model、research、meeting prep、proofread deck 等是否单独计费。
  • Data entitlements: Rogo 是否能利用客户既有金融数据订阅降低 COGS。
  • Implementation leverage: 白手套部署能否产品化,否则服务成本会随客户增长线性上升。

7.2 Cost build

成本包括:

  • LLM inference / model broker / custom finance model training;
  • professionally labeled finance data;
  • data connectors and entitlement management;
  • document ingestion、indexing、retrieval、citation;
  • Excel/PPT generation and validation;
  • security/compliance/single tenant deployment;
  • forward-deployed bankers and engineers;
  • enterprise sales。

金融 workflow 的输出质量要求高,尤其 Excel/PPT 中数字和来源要一致。Rogo 若要真正做到 client-ready,必须投入验证、source map、audit trail 和 QA,这会增加工程和服务成本。

7.3 毛利与规模化判断

积极因素:

  • 金融专业人士愿意为节省高价值时间付费;
  • 大行/资管 ACV 高;
  • workflows 可复用,随着客户积累 prompt library 和 templates;
  • Rogo 可借客户已有数据订阅避免部分内容成本;
  • 使用量大且真实,50,000+ daily queries 有 adoption 信号。

压力因素:

  • bespoke deployment 与 forward-deployed team 可能拖累 gross margin;
  • 数据许可和权限复杂;
  • client-ready Excel/PPT 失败成本高;
  • 大客户议价强;
  • 内部自建 AI 平台会持续施压。

8竞争格局

8.1 与 Hebbia

Hebbia Matrix 更像复杂文档/数据的矩阵式 analyst,适合 VDR、IC memo、contracts、expert calls。Rogo 更像投行/投资专业人士的 day-to-day agent,强调 deck、spreadsheet、reports、meeting prep。二者在金融研究和 diligence 上正面竞争:

  • Hebbia 的 UI/Matrix 可能更适合批量结构化比较;
  • Rogo 的 Felix 和 workflow library 更贴 banker deliverables;
  • Hebbia 可能服务法律/专业服务更宽;
  • Rogo 的 ex-bankers implementation 和 finance-specific model 可能更懂投行格式。

8.2 与 AlphaSense

AlphaSense 拥有 500M+ premium docs、Tegus、Canalyst、>$600M ARR 和 7,000+ customers。Rogo 的优势是 workflow execution 和 bespoke finance deployment。客户可能组合使用:

  • AlphaSense 用于市场/公司/专家内容检索;
  • Rogo 用于把材料转成 deck/model/memo 和 firm-specific outputs。

如果 AlphaSense SuperAnalyst 在 Excel/Word/PPT 和 expert call execution 上成熟,Rogo 的差异化会被压缩;如果 Rogo 在投行交付物质量上显著领先,它可以成为金融团队真正的 agent workspace。

8.4 与 Microsoft/OpenAI

Microsoft Copilot 拥有 Office/SharePoint/Teams/Excel/PPT 分发入口,OpenAI 拥有模型能力。Rogo 的防线在:

  • 金融领域 prompt/workflow;
  • data provider integrations;
  • auditable finance outputs;
  • ex-banker implementation;
  • client-ready deck/model/memo QA;
  • customized governance for financial institutions。

若 Microsoft/OpenAI 提供足够好的 Excel/PPT agent,但缺金融数据 entitlements 和投行格式,Rogo 仍有空间;若大行用 Microsoft stack 自建 finance workflow,Rogo 可能被挤压。

9团队与背景

Gabe Stengel 前 Lazard 背景很重要。投行/PE workflow 的细节非常多,从公司 strip profile、sponsor overview、meeting prep 到 pitch deck consistency,每个输出都有行业惯例。没有从业者经验的 AI team 容易把产品做成“看似聪明的聊天框”,但不能进 MD/IC/client 流程。

Rogo 官网也强调 team of former bankers and investors,并在 implementation 中让 ex-bankers/private equity investors 作为 change-management partners。这是销售优势,因为客户会更信任“lived the problem”的团队;同时也是成本挑战,因为这类人才昂贵且难规模化。

10主要风险

  1. 大行自建风险: JPMorgan、Goldman、Morgan Stanley 等都有内部 AI 平台和强数据/权限优势。
  2. 数据授权风险: Rogo 连接多个金融数据源,输出和再分发权利需要严格管理。
  3. client-ready 输出风险: Excel/PPT/memo 出错会直接影响客户交付和声誉。
  4. 服务成本风险: bespoke deployment 和 forward-deployed bankers/engineers 难以完全软件化。
  5. 竞争加剧: AlphaSense SuperAnalyst、Hebbia Matrix、Bloomberg/FactSet AI、Microsoft Copilot 都会争夺同一预算。
  6. 用户 adoption 不等于 ARR: 35,000+ users 和 50,000+ queries 是好信号,但付费规模和留存需要核验。
  7. 估值和融资节奏: Series B 到 Series D 速度快,若收入增长跟不上估值,后续融资可能承压。
