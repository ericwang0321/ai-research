公司 2026 年 7 月宣布最新融资，估值提升至约 $6B。它的核心机会来自企业和 AI-native 应用对开放模型、定制模型和多云推理的需求；主要风险是推理价格战、GPU 供应、客户自建和 Fireworks/Baseten/云厂商竞争。

Together 的定位比 Fireworks 更“全栈”。Fireworks 更像高性能 inference 和部署平台;Together 同时强调 research systems、accelerated compute、GPU clusters、AI Factory 和 post-training。这使 Together 更像面向开放模型的 CoreWeave + OpenAI API alternative + MLOps 平台混合体。对客户而言,它的购买理由是选择权、控制权和成本: 可以使用 DeepSeek、MiniMax、Kimi、GLM、Qwen、Llama、Mistral、gpt-oss 等开放模型,也可以微调、部署 custom models,并在 dedicated capacity 上控制延迟、区域和合规。

4业务模式与收费方式

4.1 Serverless Inference

Together 官方价格页显示,多数团队从 serverless inference 开始,然后在规模化后迁移到 dedicated endpoints。Serverless 按 1M tokens 计价,覆盖 chat、vision、image、audio、video、transcribe、embeddings、rerank、moderation 等类别。公开页面列出的例子包括 DeepSeek V4 Pro input $1.74/M、output $3.48/M;MiniMax M3 input $0.30/M、output $1.20/M;gpt-oss-120B input $0.15/M、output $0.60/M;Qwen3 235B A22B FP8 input $0.20/M、output $0.60/M。部分模型还有 cached input price。

Serverless 的战略意义是开发者入口和模型市场。Together 越快接入新开放模型,越能成为 AI 应用测试和迁移的默认 API。由于 OpenAI-compatible API 已成为行业事实标准,客户迁移成本主要在 eval、prompt、latency 和模型质量,而不在 SDK。

4.2 Dedicated Inference 与 Dedicated Container

当客户进入高并发和生产阶段,Together 提供 Dedicated Model Inference 与 Dedicated Container Inference。前者适合热门模型的稳定低延迟部署,后者适合 custom models 或客户自有容器。收费通常围绕 dedicated hardware、capacity commitment、throughput、region 和 support 设计,公开页面引导客户联系销售。

Dedicated 的商业价值在于把 usage revenue 变成 committed spend。对 Together 来说,客户承诺 capacity 可以提高 GPU 利用率和收入可预测性;对客户来说,它换来更稳定 SLA、更高 rate limit、更低 per-token 成本和数据/模型控制。

4.3 GPU Clusters 与 AI Factory

Together 价格页和导航将 Accelerated Compute 独立列出,包括 GPU Clusters 和 AI Factory。2026 Series C 公告称,除 $800M equity 外,Together 获得了 500MW+ compute capacity commitments,由新投资者独立资本化,以支持未来 compute growth。这说明 Together 的商业模式已经超出 API: 它需要为大型训练、post-training、inference 和企业/主权 AI 部署锁定物理 capacity。

GPU clusters 的收入可能来自:

  • 按 GPU-hour / cluster-hour 使用;
  • 多月或多年 capacity commitment;
  • AI Factory 项目的定制基础设施和托管服务;
  • 主权 AI / 私有云 / regulated deployment 的项目合同;
  • 训练、fine-tuning、evaluation 与 inference 打包。

4.4 Fine-Tuning、Evaluations 与 Model Shaping

Together 把 Fine-Tuning 和 Evaluations 放在 Model Shaping 下。其含义是: 客户不是只买一个 base model API,而是用自己的数据、反馈和 eval harness 改造模型。模型商业化的关键在 post-training: 开源模型本身是免费或低成本的,但企业需要 domain adaptation、tool calling、reasoning、vision-language、safety eval 和 deployment workflow。Together 的收入可以从一次性训练任务扩展到持续 eval/fine-tune/serve 循环。

4.5 Enterprise / Private Cloud

Together 2024–2026 的产品表达持续强调 enterprise AI platform、private cloud、sovereign AI、regulation-ready infrastructure。企业合同通常不以公开 token price 为主,而是包含:

  • committed usage / prepaid credits;
  • VPC/private cloud/on-premises deployment;
  • SSO、RBAC、audit、data residency;
  • support 和 solution engineering;
  • dedicated GPUs 或 AI Factory capacity;
  • 模型选择、benchmark、fine-tuning、eval 和 production serving。

这使 Together 更接近“开放模型平台供应商”,而非单纯 GPU 云。

技术上,Together 强调 research-to-production pipeline。2026 Series C 公告提到 FlashAttention-4 for NVIDIA Blackwell、Together Megakernel、together.compile、post-training APIs、tool calling、reasoning、vision-language models。这表明其壁垒在 kernel、compiler、inference engine、training infra 和开放模型生态组合。

Together 的平台逻辑是“全栈效率”: 模型本身、kernel、compiler、inference system、training infrastructure、hardware utilization 和上层软件都影响生产 AI 成本。若 Together 能比普通开源 serving engine 提供更高 tokens/sec、更低 P95 latency、更好 GPU utilization,就能以较低价格抢占客户且保留软件毛利。

7单位经济性 / 关键财务逻辑

Together 的单位经济性可以分为 API、dedicated 和 AI Factory 三类:

  1. Serverless API: 收入 = input/output token price × volume;成本 = GPU time + serving overhead + idle capacity + network/storage。优势是开发者自助,缺点是价格透明、竞争强。
  2. Dedicated Inference: 收入 = committed capacity 或 discounted token plan;成本 = reserved GPU 和运维。优势是可预测收入和稳定客户关系,风险是客户议价强。
  3. GPU Clusters / AI Factory: 收入 = large contract / capacity lease / managed infra;成本 = power、data center、GPU、network、financing。优势是合同大,风险是资本重、周期长。

Together 的 gross margin 取决于四个变量:

  • GPU procurement cost 与融资结构;
  • utilization,尤其是 prefill/decode 分离、batching 和多租户调度;
  • software efficiency,包括 FlashAttention、Megakernel、compiler;
  • customer pricing power,即客户是否愿意为开放模型控制权和成本节省支付平台溢价。

Series C 公告称开放模型客户常见 6x-20x 成本降低。若该节省来自模型选择而非 Together 独有软件,竞争会很快压价;若来自 Together kernel/serving/full-stack optimization,则平台可保留一部分价值。Decagon 6x cost reduction 是强案例,但需要核验迁移前后的模型、任务、质量、延迟和并发口径。

  • 开源社区和研究系统品牌;
  • 模型覆盖广,能快速响应开放模型迭代;

9团队与背景

CEO Vipul Ved Prakash 是 Together 的核心创始人和叙事输出者。公司团队背景覆盖 Stanford、MIT、University of Chicago 等研究/工程网络,并围绕开放模型、系统优化和生产 AI infrastructure 建立组织。Together 的品牌不是单纯“便宜 API”,而是“开放 AI 应该 abundant, not expensive”。这类使命叙事对吸引开源社区、研究工程师和 AI-native 客户有帮助。

2024–2025 年收购 CodeSandbox 和 Refuel.ai 也反映团队意图: CodeSandbox 增强 developer environment / code execution / European engineering presence;Refuel.ai 增强数据准备和企业 AI workflow。Together 正把自己从模型 serving 平台扩展为 AI development-to-production platform。

10主要风险

  1. 重资产风险: 500MW+ commitments 带来规模优势,也带来利用率和融资成本压力。
  2. 价格战风险: 开放模型 API 价格透明,Fireworks、Baseten、clouds 都会竞争。
  3. 模型供应风险: Together 依赖开放模型持续接近闭源模型质量。
  4. 客户自建风险: 大客户可能用 vLLM/TensorRT-LLM 自建 serving。
  5. 客户集中风险: AI-native 客户用量大,但预算和产品生命周期波动高。
  6. 云厂商挤压: AWS/Azure/GCP 用企业采购和 credits 绑定客户。
  7. 资本化风险: 若后续估值过高而毛利不够,IPO/下一轮压力上升。
  8. 合规和主权 AI 执行风险: 区域部署涉及数据、能源、政府关系和本地供应链。
