EvolutionaryScale

EvolutionaryScale 是 ESM 系列从 Meta FAIR/开源科研项目走向独立 AI biology 平台化的公司化载体，核心资产是 ESM3:一个同时对 protein sequence、structure、function 推理的 98B 参数级 generative biology model。

1一句话结论

EvolutionaryScale 是 ESM 系列从 Meta FAIR/开源科研项目走向独立 AI biology 平台化的公司化载体，核心资产是 ESM3:一个同时对 protein sequence、structure、function 推理的 98B 参数级 generative biology model。

它已经用 esmGFP 做出实验验证，模型生成的绿色荧光蛋白与最接近天然蛋白仅 58% sequence identity,研究团队将其等价描述为模拟超过 5 亿年自然演化，并于 2025 年在 Science 发表。商业化上，公司不是传统药企 pipeline,而更像生命科学基础模型平台，通过 Forge/API、AWS SageMaker/Omics/Bedrock、NVIDIA BioNeMo/NIM、open model/non-commercial license、以及未来药企/biotech discovery collaboration 变现。

最大缺口是融资、估值、收入、药企客户、milestone economics 和 API usage 没有足够公开细节。

EvolutionaryScale 的背景来自 ESM project。ESM 系列在 Meta FAIR 时期已通过蛋白语言模型展示出 structure/function representation 能力。公司化之后,ESM3 从“预测/表示蛋白”推进到“生成/设计蛋白”:用户可以给定序列、结构或功能 prompt,模型在三种模态之间联合推理,生成满足约束的新蛋白。

4业务模式与收费方式

4.1 平台模式:biology foundation model as API

EvolutionaryScale 的最直接商业化路径是 API/平台收费。官网写明所有 ESM3 models 可通过 Forge API closed beta 使用,2025 年 1 月更新中又提到 API for biological intelligence 进入 public beta,为学术和工业科学家提供有限时间免费 preview。该模式后续可自然演化为:

  • per-request / per-token / per-protein generation 收费;
  • higher-throughput enterprise API;
  • private deployment / VPC / cloud marketplace;
  • scientist workflow seat subscription;
  • model fine-tuning / custom protein design project;
  • wet-lab validated design packages;
  • pharma discovery collaboration with milestones。

4.2 云和平台分销

EvolutionaryScale 没有只依赖自有前端。官网列出:

  • AWS:ESM3 available on AWS via SageMaker,并计划进入 Omics platform 和 Bedrock;
  • NVIDIA:ESM3 will be available on BioNeMo and as NVIDIA NIM microservice,并与 NVIDIA 优化;
  • Open model:ESM3-open 代码/权重可在 GitHub 使用,non-commercial license;
  • Forge/API:直接申请访问,面向学术和工业用户。

这种分销结构对 AI biology 公司很重要:药企和科研机构通常已经在 AWS、NVIDIA BioNeMo 或内部 HPC/云环境中工作,通过 marketplace/API 可以降低采购与合规门槛。

4.3 药企合作与 milestone 逻辑

5产品与技术

5.1 ESM3:三模态 protein language model

ESM3 是一个 multi-track transformer,同时对蛋白的三个基本属性建模:

  • Sequence:氨基酸序列;
  • Structure:三维结构离散化后的表示;
  • Function:功能关键词、注释和相关信号。

模型训练目标类似 masked language modeling:对蛋白的 sequence、structure、function 信息做 tokenization 和 masking,再预测被遮盖部分。因为三种属性在同一模型中联合训练,ESM3 可以接受任意组合 prompt,生成满足序列、结构或功能约束的新蛋白。

官网披露的规模:

  • 最大模型:98B parameters;
  • 训练数据:2.78B natural proteins sampled from organisms and biomes;
  • unique tokens:771B;
  • 训练计算:约 10^24 FLOPs / 1 trillion teraflops 口径。

5.2 esmGFP:实验验证的核心证据

ESM3 最重要的公开验证是 esmGFP。研究团队用 GFP 作为目标,让模型生成新的绿色荧光蛋白:

  • 第一轮测试 96 个生成候选,发现 B8 等可发光蛋白;
  • 继续以 B8 为起点做 chain-of-thought 生成,再测试 96 个蛋白;
  • 得到 esmGFP,亮度接近天然 GFP;
  • esmGFP 与最接近的天然 fluorescent protein 仅 58% sequence similarity,229 个氨基酸中有 96 个突变;
  • 研究团队估算相当于模拟超过 500 million years evolution;
  • 论文于 2025 年在 Science 发表。

这类证据对 AI biology 公司非常关键,因为它不是纯 benchmark。AI 药物发现常被质疑“模型分数高但实验不成”。esmGFP 证明 ESM3 至少能在一个明确蛋白家族任务上提出远离天然序列空间且仍具备功能的设计。

5.3 应用方向

官网列出的方向包括:

  • 设计用于 carbon capture 的 protein;
  • 设计降解 plastic 的 enzymes,如 PETase active-site scaffold;
  • 设计 new medicines,包括 therapeutic proteins/antibodies;
  • 从 sequence/structure/function prompt 组合生成候选;
  • 利用实验或已有数据反馈对模型 generation 做 alignment。

5.4 ESM3-open 与 license

官网称 ESM3-open 是小但强的模型,权重和源代码在 GitHub 上以 non-commercial license 发布。该策略类似 AI biology 平台常见的“双轨制”:

  • 小模型开放给学术社区,扩大引用和生态;
  • 大模型/API/企业部署通过商业 license 变现;
  • 用 responsible development framework 限制生物安全风险。

这与 Mistral/Meta 的 open-weight 策略不同:蛋白设计涉及合成生物学和 biosecurity,因此 license 与访问控制更严格,不能简单等同通用 LLM 开源。

5.5 ESM Cambrian

官网首页在 2026 年显示 “Introducing ESM Cambrian”,称其为 parallel model family to generative flagship ESM3,用于 protein sequence modeling 和 representation learning,并宣称 performance/efficiency 达到新 frontier。ESM Cambrian 的商业意义是补齐 ESM3 之外的 representation layer:药企不仅需要生成蛋白,也需要对已有序列、变体、功能和 disease relevance 做 embedding、ranking 和 downstream prediction。

7单位经济性 / 关键财务逻辑

EvolutionaryScale 的财务逻辑与传统 biotech 不同,也与通用 LLM API 不同。

7.1 API / platform economics

收入可能来自 protein generation、embedding、ranking、structure/function-conditioned design 和 workflow seats。成本主要是模型推理、数据库/存储、云分发、scientist support 和安全审核。若客户主要通过 AWS/NVIDIA 使用,还会有 marketplace revenue share。

关键变量:

  • 每个 protein design job 的 compute cost;
  • 每个客户项目需要多少生成轮次和实验反馈;
  • 大模型是否必须调用,还是小模型/ESM Cambrian embedding 就足够;
  • 是否需要 EvolutionaryScale scientist-in-the-loop 支持;
  • 免费 public beta 何时转付费。

7.2 Drug discovery collaboration economics

若与药企做项目,收入通常不会只按 API token 计费,而是:

upfront + annual platform fee + FTE/research support + preclinical/clinical milestones + royalties

但这个逻辑只有在具体项目和 IP 权属公开后才能量化。AI biology 公司最重要的单位经济性是“每个 discovery program 产生可验证 hit/lead 的成本和时间”,不是传统 LLM 的 cost per 1M tokens。

7.3 Wet-lab validation economics

esmGFP 的验证说明模型能生成远离天然空间的功能蛋白,但药物发现的经济性还要看:

  • hit rate:生成多少候选能通过体外实验;
  • optimization rounds:需要几轮设计-合成-测试;
  • assay cost:每轮实验成本;
  • translational relevance:体外功能是否转化到细胞/动物/临床;
  • IP defensibility:生成蛋白是否可申请专利并绕开现有序列空间。

7.4 为什么估值不能简单按 biotech pipeline 看

EvolutionaryScale 更像 platform company。它可以同时服务多个 therapeutic areas、industrial enzymes、research tools 和 cloud workflows。如果商业化成功,收入会来自平台 license 和多项目 milestones;如果失败,则可能停留在科研工具和论文影响力。

8.2 价值链位置

EvolutionaryScale 位于药物发现价值链上游:

基础模型 → 候选蛋白设计 → 合成/表达 → in vitro assay → optimization → preclinical → clinical

它的价值不是替代药企全部研发,而是提高 early discovery 的搜索效率。若模型能把有效候选从极低 hit rate 提升到可工程化水平,就能改变药物发现的前端经济性。

8.3 护城河

  • ESM legacy:ESM 系列在蛋白语言模型领域有长期学术积累;
  • 数据规模:2.78B proteins、771B tokens;
  • 模型规模:98B 最大模型;
  • 实验验证:esmGFP 不是纯 in silico 结果;
  • 云分发:AWS/NVIDIA 合作降低企业采用门槛;
  • responsible development:蛋白设计需要安全治理,成熟框架本身是进入药企/政府客户的前提。

9团队与背景

Alexander Rives

Alexander Rives 是 ESM 项目的核心人物之一,在公开采访和公司文章中担任 EvolutionaryScale co-founder/chief scientist 角色。他长期研究 protein language models,ESM3 文章也由其团队主导。

前 Meta ESM 团队

EvolutionaryScale 的关键优势来自原 Meta FAIR 的 ESM 研究积累。Meta 时代的 ESM-1b、ESM-2、ESMFold 等已经在蛋白结构、功能、变体效应和抗体工程研究中广泛引用。公司化后,这些经验转化为更大规模、更偏生成式的 ESM3。

科学与工程复合能力

AI biology 公司必须同时拥有:

  • ML scaling 和 transformer 训练能力;
  • protein biology 和实验设计理解;
  • cloud/API 产品能力;
  • biosecurity/responsible development 经验;
  • 与药企研发流程对接的 BD 能力。

公开资料能证明其 ML/protein research 强,但商业 BD 和药企合作深度还需要更多外部证据。

10主要风险

  1. 商业化证据不足:没有公开 ARR、API 收入或药企合同金额。
  2. 融资口径不够透明:常见 $142M seed 口径需要通过数据库/投资方文件继续交叉核验。
  3. 从 GFP 到药物的转化风险:esmGFP 是强验证,但 therapeutics 对 potency、specificity、immunogenicity、PK/PD 要求更高。
  4. 实验闭环成本:模型生成只是第一步,合成、表达、assay 和优化仍昂贵。
  5. 生物安全风险:蛋白设计模型可能被用于有害分子设计,访问控制和 license 至关重要。
  6. 竞争风险:AlphaFold/Isomorphic/Generate/Cradle/Chai/Profluent 都在争夺药企预算。
  7. 云平台依赖:AWS/NVIDIA 分发降低门槛,也可能压缩平台议价权。
  8. IP 风险:AI 生成蛋白的专利归属、训练数据来源和 freedom-to-operate 复杂。
  9. 模型泛化风险:在 GFP 家族成功不代表在抗体、酶、膜蛋白或多蛋白复合体上同样成功。
  10. 组织/平台变化风险:ESM API 与 Biohub/GitHub 生态的关系需要跟踪,避免误判公司资产边界。
