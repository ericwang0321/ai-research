Braintrust

Braintrust 是生产 AI 应用的评测和可观测性平台。它把 traces、datasets、experiments、LLM-as-judge、code/human scoring、online monitoring、Topics pattern discovery、Loop agent 和 Brainstore 数据层放在同一套质量闭环里。

1一句话结论

Braintrust 是生产 AI 应用的评测和可观测性平台。它把 traces、datasets、experiments、LLM-as-judge、code/human scoring、online monitoring、Topics pattern discovery、Loop agent 和 Brainstore 数据层放在同一套质量闭环里。

它解决的是 AI 产品质量会漂移、退化且难以解释的问题，而不是给传统 APM 加一个 AI 插件。截至 2026 年 7 月 6 日，官方 pricing 显示 Starter $0/month、Pro $249/month、Enterprise custom。投资重点是 Braintrust 能否成为企业 AI 应用上线前后必备的质量基础设施。

Braintrust 的公司叙事是"AI fails differently than normal software"。传统软件的可靠性主要看请求成功率、latency、error rate、CPU/memory、logs/traces;AI 应用的失败则可能是回答不符合业务口径、幻觉、检索错上下文、tool call 顺序错、agent 走错轨迹、prompt 改动导致某些用户段退化、LLM-as-judge 本身漂移、成本突然上升。仅靠 Datadog/New Relic 的系统指标看不见这些质量问题。

Braintrust 的产品思路是把生产 trace 转化为 evaluation asset: 从线上日志中抽样,自动发现 Topic/issue/sentiment pattern,把失败样本沉淀为 golden datasets,用 scorers 做 offline/online eval,再在 release 或 prompt/model 更新前设 quality gates。这样 AI 产品迭代不再只靠人工 eyeballing 或 PM 试用,而是进入类似 CI/CD 的质量工程。

Braintrust 的 pricing 有两个特点。第一,用户数、projects、datasets、experiments、playgrounds 不按 seat 限制,降低组织协作阻力。第二,真正 monetization 发生在 processed data、scores、Topics tokens、retention/export/security 和 enterprise deployment 上。这符合 AI observability 的数据密集属性: trace 越多、在线打分越多、自动聚类越多,费用越高。

4.2 Processed data

Processed data 是 Braintrust 的基础用量维度。AI traces 通常很大: prompt、response、tool calls、retrieval chunks、intermediate state、metadata、scores、attachments、conversation history 都会进入日志。官网 Series B 博客提到 long-running multi-step agents 可能每次交互产生 hundreds of megabytes 的数据。Starter 包含 1GB 后 $4/GB,Pro 包含 5GB 后 $3/GB,Enterprise custom。

这类似 Datadog 的 log ingestion,但数据形态更复杂。Braintrust 为此推出 Brainstore,声称传统数据库不适合复杂 AI traces,需要专为 nested AI observability data 设计的数据层。Processed data 毛利取决于压缩、索引、查询、冷热分层和 retention 设计。

4.3 Scores / evals

Scores 是 scored outputs from LLM-as-a-judge、autoevals 或 custom code scorers。Starter 10k/month 后 $2.50/1k;Pro 50k/month 后 $1.50/1k;Enterprise custom。这个维度直接映射 AI quality work: 每次 offline experiment、online sampled trace、release gate、human review 或 regression test 都会产生 scores。

Braintrust 的收入随客户质量覆盖率提升而增长。一个 AI support agent 如果每天处理大量 conversation,客户可能只抽样 1%-10% 做自动评分;一旦某些风险场景需要更高覆盖率,score usage 会明显增加。Braintrust pricing calculator 用 monthly tokens 和 scoring percentage 推算 sampled traces 和 quality checks,说明公司在教育客户把生产 traffic 转成质量信号。

4.4 Topics token rates

Topics 是 2026 年重点产品,用于在生产 traces 中自动发现、分类和聚类 pattern。定价显示 Topics included credits 后按 $0.06/mtok input、$0.40/mtok output。Topics 的商业意义是把 observability 从被动 dashboards 变成 active observability: 系统自动告诉团队哪些问题正在出现、哪个客户段/任务/情绪/issue 在变化,并推动生成 eval datasets。

如果 Topics 成功,Braintrust 不只是存储和展示 traces,而是参与改进循环。这会提高 switching cost,因为 topic taxonomy、facet、datasets、scores 和 release gates 会沉淀成客户的 AI quality knowledge base。

4.5 Enterprise contract

Enterprise 价值点包括 custom retention/export、RBAC、SAML SSO、DPA、BAA/HIPAA、uptime SLA、guaranteed service level agreements、shared Slack、premium support、on-prem/hosted/hybrid deployment。官网首页也强调 SOC 2 Type II、GDPR、SSO、RBAC、HIPAA 和 hybrid deployment。

AI observability 触及高度敏感数据: 用户输入、内部文档、RAG chunks、tool outputs、业务决策、模型回答、可能的 PII/PHI。金融、医疗、教育、企业 SaaS 客户常常不能把完整 traces 放进普通多租户 SaaS。因此 Braintrust 的 hybrid/on-prem data plane 是 enterprise monetization 的关键。

5产品与技术

5.1 Observability

Braintrust Observability 提供生产 traces、tool calls、prompts、responses、latency、cost、quality metrics、custom views、annotation interfaces、dashboards 和 searchable logs。首页强调客户可以 inspect every trace and tool call, search across millions of logs, track latency, cost, and quality in real time。

AI observability 与传统 observability 的区别在于"质量"是第一等对象。错误不一定表现为 exception;模型可能成功返回 JSON,但内容错、引用错、风格错、业务规则错。Braintrust 把 trace 与 score、dataset、experiment 绑定,让每次观察都能进入 eval loop。

5.2 Evals

Braintrust Evals 支持定义 datasets、运行 experiments、比较 prompts/models、使用 LLM/code/human scorers、构建 offline regression tests 和 online scoring。官网描述为 define what good looks like before you ship。这个理念很关键: AI 产品应先定义评价标准,再开发/上线,而不是上线后凭用户反馈救火。

客户案例显示 Coursera 用 Braintrust 为 Coursera Coach 和 AI grading 建立评估流程,包括 clear criteria、targeted datasets、heuristic/model-based scorers、online monitoring/offline testing/playground prototyping。案例给出的业务指标包括 Coursera Coach 90% learner satisfaction、AI grading 带来约 45x more feedback 和 16.7% increase in course completions within a day of peer review。

5.3 Automation / Topics

Automation 包含 Topics、online scoring、quality gates、alerts。Topics 会自动从生产数据中发现 patterns,并允许 custom facets,如 use case、customer segment、compliance、tone。它的战略目标是把"看日志"变成"系统主动发现问题和改进机会"。

2026 年 6 月 Topics GA 说明 Braintrust 的方向从 early eval tooling 向 production AI ops 扩展。随着 agent traces 变长,人工逐条看不现实;自动聚类和 pattern discovery 成为必要能力。这里 Braintrust 与 LangSmith Engine、Arize Phoenix/AX、Humanloop、Patronus、Galileo、OpenAI Evals-like tooling 竞争。

5.4 Loop agent

Loop agent 是 Braintrust 内置的 AI agent,可运行 evaluations、生成 test cases、迭代 prompts。首页描述为"AI that helps you improve AI"。商业上,Loop 让 Braintrust 从工具变成 semi-automated AI quality engineer: 用户描述优化目标,Loop 生成 prompts、scorers、datasets。

如果 Loop 可靠,它能显著提高使用频率和平台粘性;如果不可靠,可能变成 demo 功能。关键指标是 Loop 生成的 test/scorer/prompt 被采纳率、减少人工 eval engineering 时间、是否能在大型客户生产流程中通过 review。

5.5 Brainstore

Brainstore 是 Braintrust 自研的 AI observability database。公司称 AI traces large and nested,传统数据库难以处理复杂度。2025-2026 博客包括 Brainstore is now on by default、Brainstore makes AI observability at scale possible、How Brainstore works 等,说明底层数据架构是护城河重点。

Brainstore 的价值在于:

  • 快速写入高体量 nested traces;
  • 对 prompt/response/tool call 进行全文搜索和结构化查询;
  • 支持 custom trace views 和 facets;
  • 在 retention/export 下控制存储成本;
  • 支持 hybrid data plane;
  • 服务 Topics/Loop 这样的上层智能功能。

如果 Brainstore 只是内部存储实现,护城河有限;如果它在查询速度、成本和 AI-native schema 上显著优于通用 log store,则能支撑 enterprise scale。

5.6 MCP、SDK 与 framework agnostic

Braintrust 支持 Python、TypeScript、Go、Ruby、C# 等 SDK,并强调 works with any stack, no framework lock-in。MCP server 允许 coding agents 从 IDE 查询 logs、运行 evals、更新 prompts。这个定位使 Braintrust 不绑定 LangChain/LlamaIndex/OpenAI/Anthropic 任一框架,有机会成为中立 AI quality control plane。

中立性是双刃剑: 它扩大 TAM,但也意味着任何大平台都可以在自身生态内复制部分功能。Braintrust 必须通过跨框架、跨模型、跨组织协作和 enterprise data deployment 来保持独立价值。

7单位经济性 / 关键财务逻辑

Braintrust 的财务逻辑类似"AI-native Datadog + test platform",但比传统 APM 更依赖数据和模型调用:

  1. Ingestion/storage margin: processed data GB 收入要覆盖写入、压缩、索引、查询和 retention。AI trace 大而复杂,Brainstore 若降低单位存储和查询成本,毛利更好。
  2. Scoring margin: score 可能来自 code scorers、LLM-as-judge 或 human review。LLM scorer 有第三方模型成本;code scorer 成本低;human review 是协作功能而非纯自动成本。
  3. Topics token margin: Topics 使用 LLM 对 logs 聚类/分类,按 token 收费。若可用小模型或自研 routing 降成本,毛利可改善。
  4. Enterprise ACV: 安全、hybrid、custom retention、premium support 和 SLA 是高 ACV 来源,也带来 sales/support 成本。
  5. Unlimited users 策略: 不按 seat 收费能促进跨团队采用,但必须通过 usage/enterprise 把价值回收。
  6. Expansion motion: 客户从一个 chatbot eval 扩展到所有 AI features、agents、RAG、code assistant、support automation,processed data 和 scores 会自然增长。

核心问题是 Braintrust 能否成为 system of record for AI quality。如果 traces、datasets、scores、experiments 和 release gates 都沉淀在 Braintrust,切换成本会高;如果客户只是把它当临时 eval dashboard,用量和留存会弱。

8.1 与 LangSmith

LangSmith 与 Braintrust 最直接。LangSmith 绑定 LangChain/LangGraph 开源生态,从 agent development、observability、evaluation 到 deployment 形成闭环。Braintrust 的差异是 framework agnostic、AI quality platform 叙事更集中、Brainstore/Topics/Loop 强调生产 trace intelligence。对使用 LangGraph 的团队,LangSmith 入口更自然;对多框架、多模型、需要中立 eval/observability 的企业,Braintrust 更容易进入。

8.2 与传统 observability

Datadog/New Relic/Honeycomb 有企业渠道和日志预算,但 AI-specific eval datasets、LLM scoring、prompt/model experiments、trace-to-dataset、quality gates 不是传统强项。Braintrust 的窗口期在于传统厂商产品补齐前成为 AI engineering 默认工具。长期这些厂商可能收购或复制 AI observability,压缩 Braintrust 独立空间。

8.3 与模型厂商

OpenAI、Anthropic、Google、AWS、Azure 都会提供 evals、traces、monitoring。模型厂商优势是离 inference 最近,数据天然在平台内;劣势是多模型/多框架中立性不足。企业通常不会只用一个模型,Braintrust 的跨 provider 价值在这里成立。但若某个模型平台成为企业 agent runtime 默认入口,Braintrust 需要强集成才能避免被边缘化。

9团队与背景

Ankur Goyal 在 Series B 公告中解释,他在 Impira 和 Figma 都经历过内部建设 eval tools 的痛点,因此创立 Braintrust。这个背景重要: Braintrust 不是从传统 observability 复制概念,而是从 AI product teams 实际质量控制问题出发。

团队能力需要关注:

  • 是否具备 database/log infra 能力,支撑 Brainstore;
  • 是否懂 AI product evaluation,而不只是日志工具;
  • 是否能服务 PM、engineer、ML/AI engineer、product ops 等跨职能用户;
  • 是否能处理 enterprise security/hybrid deployment;
  • 是否能用 customer obsession 驱动高 ACV 采购和扩展。

ICONIQ 投资人 Matt Jacobson 在公告中强调 customer obsession,这与 Braintrust 客户案例密集发布相一致。公司官网和博客更新频率很高,2025-2026 年围绕 evals、agents、Brainstore、Topics、privacy、MCP、model comparison 输出大量内容,显示团队在 product education 上投入较重。

10主要风险

  1. 平台内置风险: OpenAI/Anthropic/Google/AWS/Azure/LangChain 都会提供 traces/evals,客户可能优先用已有栈。
  2. 品类边界风险: AI observability 可能被传统 observability 厂商作为 feature bundle,削弱独立采购。
  3. 数据成本风险: Agent traces 数据量大,若 retention 和查询成本高,processed data 毛利承压。
  4. LLM judge 可靠性风险: 自动评分本身会漂移、偏差和被 prompt attack,需要强校准和 human review。
  5. 数据敏感风险: traces 可能包含 PII、PHI、内部知识、客户信息和 prompt secrets,安全事故会严重损害信任。
  6. ROI 证明风险: Evals 很重要但容易变成"质量团队工具";Braintrust 需要持续证明能提升上线速度、降低事故、改善指标。
  7. 早期融资信息缺口: 早期轮次、估值、ARR 和付费客户数不足,估值判断难。
