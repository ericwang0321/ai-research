Manus / Butterfly Effect

Manus 是 2025 年最受关注的通用 Agent 公司之一。它的核心不在底层大模型，而在多模型调用、云端电脑/浏览器执行、外部工具连接和可交付工作流。

1一句话结论

Manus 是 2025 年最受关注的通用 Agent 公司之一。它的核心不在底层大模型，而在多模型调用、云端电脑/浏览器执行、外部工具连接和可交付工作流。

Manus 可以从一个自然语言任务出发，完成研究、网页/应用搭建、设计、幻灯片、邮件、Slack、Google Workspace、Shopify、Airtable 等跨工具流程。公开资料显示其 2025 年 4 月由 Benchmark 领投约 $75M Series B、估值约 $500M，年底又出现 Meta $2B+ 收购/交易及中国监管审查线索。投资关键是它能否把通用 Agent 热度转成稳定、可控、可收费的工作流自动化。

Manus 的定位不能和所有“agent”混在一起。它不是 AI search，虽然能做研究；不是 coding IDE，虽然能生成网站/应用；不是企业知识搜索，虽然能接 Google Drive/Notion/Airtable；不是 RPA，虽然能做浏览器动作；也不是结果收费 marketplace。更准确说，Manus 是面向个人和小团队的通用任务执行 agent + 云端工作台 + connector automation layer。其产品叙事是“Less structure, more intelligence”：用户不需要先搭 workflow chart，而是描述目标，Manus 自动规划、搜索、调用工具、写文件、生成应用/网页/幻灯片，并把结果托管或交付。

4业务模式与收费方式

4.1 订阅和 credits

公开资料和产品页面/百科汇总显示 Manus 采用 credits + subscription:

计划价格/额度口径功能含义
Freeone free daily task，约等于 300 credits；一次性 1,000 credits bonus 线索低门槛获客、控制成本
Starter$39/month，3,900 credits/month，最多 2 个并发任务轻度个人用户
Pro$199/month，19,900 credits/month，最多 5 个并发任务，beta features重度个人/专业用户
Team$39/seat/month，5 seat minimum，约 $195/month，19,500 shared credits，dedicated infrastructure小团队协作
Business/APIcustom/API/Team plan 入口面向企业、开发者和嵌入式 workflow

credits 机制适合 general agent，因为不同任务成本差异极大: 简单问答、网页调研、幻灯片生成、浏览器操作、应用构建、外部 connector 调用、图片/音乐生成、托管网站，都对应不同模型和工具成本。Manus 如果只用固定 seat，很难处理 heavy user；如果纯 usage，又会降低转化。Starter/Pro/Team 是对成本和易用性的折中。

4.2 Browser/action automation

Manus 官网和博客列出 Manus browser operator、Preferred Browser、My Computer、Cloud Computer、Google Workspace、Gmail/Calendar、Slack、Zoom、Shopify、HubSpot、Airtable、Notion、Similarweb、Meta Ads Manager、Instagram 等 connectors。这说明其商业化正在从“通用 agent demo”转向“workflow automation”: 帮用户跨应用搜索、更新、报告、建站、营销、会议纪要、CRM follow-up、社媒/广告素材、店铺搭建。

收费逻辑上，这类产品不是传统按结果收费，而是:

  • 订阅包含 credits 和并发额度。
  • Team/Business 按 seat 和共享 credits。
  • API 面向开发者或企业嵌入。
  • connector/workflow 产生更高消耗，可能推动 Pro/Team 升级。
  • 托管网站、custom domains、app sharing/testing、cloud computer 可能带来增值服务。

4.3 与 OpenAI/Anthropic/Google/Meta 的关系

Manus 不以训练 frontier foundation model 为核心。公开资料多次强调其依赖 American AI models / existing LLMs / multi-model orchestration。这带来双重影响:

  • 资本效率更高: 不必像模型公司投入数十亿美元训练。
  • 护城河更脆弱: OpenAI Operator/Deep Research、Anthropic Computer Use、Google Gemini/Astra/Mariner、Meta AI 都可用更强模型复制一部分 agent workflow。

Meta 相关交易报道进一步说明，Manus 的价值在于产品、团队、workflow data 和 agent orchestration，而不是底层模型权重。若 Meta 能整合 Manus 到 Meta AI/WhatsApp/Instagram/Facebook business tools，Manus 的 distribution 想象很大；但中国监管对技术外流和交易合规的审查，使资本化状态复杂。

5.2 技术架构: kitbashing / orchestration 而非模型训练

公开报道和行业分析将 Manus 描述为对现有模型和工具的“extreme repackaging”或 “kitbashing”。这并非贬义，应用层 agent 的工程难点不在单次回答，而在:

  1. Task planning: 将模糊目标拆成可执行步骤。
  2. Tool selection: 选择浏览器、文件、API、生成器、连接器。
  3. Context management: 长任务中保持状态、记忆、项目上下文。
  4. Execution monitoring: 处理网页变化、错误、权限、验证码、失败重试。
  5. Artifact generation: 输出表格、PPT、网页、应用、报告、图片、代码。
  6. Hosting and handoff: 把结果部署、分享、复制、测试、连接域名。

2025-07 官方博客 “Context Engineering for AI Agents” 也体现了其技术重心: 不是追求最大模型，而是让 agent 在真实任务中持续管理上下文、工具和执行轨迹。

5.3 产品效果

Manus 2025 年 3 月发布后迅速走红，demo 展示了简历筛选、股票分析、游戏/网页/报告生成等任务，视频 20 小时内百万级观看量；邀请码在 Xianyu/Goofish/eBay 等平台被炒至高价，媒体报道从 $1,000 到数千美元甚至人民币 50,000-100,000 的挂牌价格。热度说明市场对 general agent 有强烈想象，但早期用户也反馈 error messages、endless loops、factual inaccuracies、crashes。这正是 general agent 的核心难点: 演示可以惊艳，稳定可控的生产级执行很难。

2025 年下半年和 2026 年，Manus 的产品发布从“会做很多任务”转向更具体的 workflows: Wide Research、Browser Operator、Projects、Google Drive/Slack/Notion/Shopify/Similarweb connectors、Custom Domains、Editable Slides、Meeting Minutes、Project Skills、Cloud Computer、Preferred Browser、Scheduled Tasks、Zoom Connector、Canva/Mobbin/HubSpot/Airtable 等。这说明公司意识到通用 agent 必须落到可重复场景，否则留存难以证明。

7单位经济性 / 关键财务逻辑

7.1 成本结构

Manus 的成本比普通 chat app 更重:

  • 多模型 token 调用，可能包括 OpenAI/Anthropic/Google/open-source 模型。
  • 浏览器/云电脑运行成本。
  • 文件、图片、幻灯片、音乐、网页/应用生成的生成成本。
  • 外部 API/connector 调用成本。
  • 托管网站、custom domain、app sharing/testing、storage。
  • 任务失败、重试、长时间执行导致的浪费。

7.2 Credits 为什么重要

Credits 机制本质上是把复杂成本抽象成用户可理解的预算。一个“写一段邮件”的成本和“做完整竞争分析、生成 deck、建站并托管”的成本差异巨大。Starter/Pro/Team 的 credits + concurrency limits 帮助公司:

  • 限制 heavy users 对毛利的冲击。
  • 让用户根据任务复杂度自我筛选。
  • 为 Team/Business 提供 shared quota。
  • 给 API/Enterprise 留出 custom pricing 空间。

7.3 Revenue run-rate 与毛利问题

若 2025 年 12 月 $125M run-rate 口径成立，而 Series B 后估值 $500M，则交易前收入倍数很低；若 Meta $2B+ 交易成立，则约 16x run-rate，仍低于许多 AI 应用高估值倍数。这说明市场可能更看重产品速度、agent data 和战略价值，而不仅是当前收入。但由于 general agent COGS 和失败率高，gross margin 是最关键的数据缺口。一个能收费 $199/月但频繁跑长任务的用户，未必比 $20/月 coding assistant 更赚钱。

8.2 Coding/app builder 侧竞争

Manus 能生成网页和应用，但并非专业 coding IDE。它与 Cursor、Cognition、Replit、Lovable 的边界是:

  • Cursor: 专业开发者 IDE，面向已有 repo 和日常编码。
  • Cognition: autonomous software engineer，面向工程任务和企业 codebase。
  • Replit/Lovable: prompt-to-app、托管、非专业开发者建应用。
  • Manus: 更泛化，既可建站/应用，也可做研究、设计、幻灯片、CRM、营销、会议、浏览器动作。

8.3 护城河

  • 产品想象力和品牌热度: 2025 年 launch 形成“general agent”心智。
  • workflow breadth: connectors 和垂直 templates 快速扩张。
  • 云端执行环境: browser/operator/cloud computer 让 agent 能交付 artifacts。
  • 全球化定位: 新加坡主体、英文产品、Meta 交易，使其摆脱纯中国本土应用标签。
  • 团队消费产品经验: Monica/Peak Labs 背景擅长 consumer AI 产品增长。

8.4 脆弱点

  • 上游模型依赖强，基础能力容易被 OpenAI/Anthropic/Google 复制。
  • 通用 agent 成功率天然不稳定，用户预期极高。
  • 监管和地缘政治风险远高于普通美国 AI 应用公司。
  • 工作流太宽会导致产品定位发散，难形成单一高留存场景。
  • Connector 生态需要持续维护，API 变化会导致 agent 失败。

9团队与背景

公开报道将 Manus 团队与 Monica、Butterfly Effect、Peak Labs 联系在一起。肖弘被报道为创始人之一；季逸超 Peak Ji 早年创立 Peak Labs，曾入选 Forbes China 30 Under 30；张涛被报道为 founding team/product director。团队特点是 consumer product、增长、AI 应用集成和跨境产品经验强，而非 foundation model research lab。

这与 Manus 的产品选择一致: 不训练自有大模型，而是把多个模型、浏览器、文件、设计、应用、连接器和托管能力组装成通用任务执行平台。团队的优势是快、会做产品传播、懂用户场景；短板是底层模型和基础设施不完全自主，且跨境主体/中国历史业务带来监管复杂度。

10主要风险

  1. 监管/地缘政治风险: 中美对 AI 技术、数据、投资和收购审查都可能影响 Manus。
  2. 模型依赖风险: 若 OpenAI/Anthropic/Google 限制 API 或推出更强原生 agent，Manus 差异化受压。
  3. 通用 agent 可靠性: 早期用户反馈 loop、错误、崩溃，生产级稳定性仍需证明。
  4. 毛利压力: 长任务、浏览器、云电脑、多模态生成和 connector 调用成本高。
  5. 定位发散: 研究、建站、slides、CRM、会议、营销、设计、浏览器动作都做，可能分散产品资源。
  6. 隐私和权限: Gmail、Calendar、Slack、Drive、Ads、Instagram 等 connectors 涉及敏感数据和权限滥用。
  7. 资本化不确定: Meta 交易若被要求撤回或长期悬置，员工/客户/投资人预期都会受影响。
  8. 竞争复制: 大厂有模型、分发和生态，Manus 的 workflow breadth 未必形成深护城河。
