Tavily

Tavily 是面向 AI Agent 的实时网页访问 API。它把 search、extract、crawl、map 和 research 做成开发者可调用的接口，让 Agent 能访问实时网页、抽取结构化内容、生成引用，并降低 PII 和 prompt injection 风险。

1一句话结论

Tavily 是面向 AI Agent 的实时网页访问 API。它把 search、extract、crawl、map 和 research 做成开发者可调用的接口，让 Agent 能访问实时网页、抽取结构化内容、生成引用，并降低 PII 和 prompt injection 风险。

截至 2026 年 7 月 6 日，Tavily 官网披露 2M+ developers、300M+ monthly requests、99.99% uptime、180ms p50 on /search、billions of pages crawled/extracted。它与 Exa 是直接竞品，也会和 Perplexity API、Brave Search API、Google/Bing Search API、Firecrawl、Linkup、SerpAPI 等竞争。

Tavily 的 credit 模型让客户不必理解每个 endpoint 的底层成本，但对投资分析也意味着需要知道不同 endpoint credit burn。Search、Extract、Crawl、Research 的成本结构差异很大。Enterprise custom rate limits 和 API call commitments 是支撑 gross margin 的关键。

4.2 产品收费与 value metric

Tavily 的 value metric 是“agent 可安全访问 web 的次数/深度”。Search 是实时检索，Extract 是从网页抽取内容，Crawl/Map 是站点级探索，Research 是多步研究。客户用量会随 agent 运行频次增长，而不是随 seat 线性增长。因此 Tavily 比 GoLinks/GoSearch 更像 usage-based API infra，比 Dify/LangChain 更少 seat/subscription，更靠调用量和 enterprise minimums。

4.3 Enterprise contract

Enterprise 合同的价值在四个方面:

  • 更高 rate limits 和 custom API calls；
  • 安全、隐私、zero data retention、PII redaction、prompt injection protection；
  • support/SLAs；
  • 可能的 private deployments、custom policies、采购条款和集成服务。

Tavily 加入 Nebius 后，enterprise sales 可以借 Nebius AI cloud 客户资源扩张；但也要看 Nebius 是否把 Tavily 打包为云服务，还是保持独立 API 品牌。

5产品与技术

5.1 Search API

Search API 是 Tavily 的核心。官网称它为 one secure API for real-time web access，支持 agent grounding、live web retrieval、structured chunks、latency control。首页展示 180ms p50 /search 和 300M+ monthly requests。对 agent 来说，搜索结果必须在可接受 latency 内返回，否则会拖慢 multi-step workflow。

5.2 Extract / Crawl / Map

Docs 展示 Tavily SDK 支持 search、extract、crawl、map 和 research。Extract 将网页正文抽取给模型，Crawl 根据 instructions 爬取站点，Map 探索站点结构。这些能力让 Tavily 不只是 SERP API，而是 web data processing layer。

5.3 Research API

Research API 用于多步、引用型研究。Tavily 2026 年博客多次提到 /research 正在成为现代 AI 系统 building block，并展示 product research agent、competitive intelligence、Fieldway 5-hour manual cycle to 25-minute agent 等案例。Research API 的战略意义是提高单次请求价值，从低价 search credits 上移到高价值 research jobs。

5.4 安全与治理

Tavily 首页强调 built-in safeguards: 请求经过 security、privacy、content validation layers，可以 block PII leakage、prompt injection 和 malicious sources。IBM partnership 页面也强调 zero data retention、PII redaction、prompt injection detection/mitigation。对于企业 agent，web 是不可信输入源，prompt injection 和恶意网页会影响 agent 行动。Tavily 若能在安全上领先，将比单纯搜索质量更有 enterprise 差异。

5.5 开发者生态

Tavily 提供 Python SDK、JavaScript SDK、cURL、docs、community、certification、events、use cases，并与 LangChain、Groq、OpenAI/Anthropic/Groq provider integrations、IBM watsonx、Databricks MCP Marketplace、JetBrains、Arcade.dev、x402 等生态集成。它的 GTM 更像 developer infrastructure: 先进入开源/agent framework，再转企业。

5.6 产品效果

官网公开指标非常强:

  • 2M+ developers using Tavily；
  • 300M+ monthly requests handled；
  • 99.99% uptime；
  • 180ms p50 on /search；
  • billions of pages crawled/extracted without downtime；
  • trusted by Writer、MongoDB、Groq、LangChain、Monday、AWS、Mastercard、Ironclad、IBM、BCG 等。

这些是 agent search API 公司最重要的 traction 指标。但需要注意: 2M+ developers 可能包含 free/community users，300M+ monthly requests 需要拆分 free vs paid、search vs extract/research、top customers concentration。

7单位经济性 / 关键财务逻辑

Tavily 的单位经济性围绕 credit 经济展开:

  • Revenue per credit: PAYG $0.008/credit 是基础价格，但不同计划和 enterprise 折扣会改变实际 ARPU。
  • Cost per web operation: Search、Extract、Crawl、Map、Research 的成本不同；Research 多步任务可能消耗更多 crawling、retrieval、LLM processing。
  • Cache / indexing efficiency: 300M+ monthly requests 下，缓存和索引效率决定毛利。
  • Security overhead: PII redaction、prompt injection detection、malicious source filtering 会增加计算成本，但也支撑 enterprise price。
  • Nebius infra synergy: 加入 Nebius 后，算力、存储、网络和全球部署成本可能下降，或至少获得更稳定供应。
  • Enterprise minimums: 自定义合同可以通过 minimum commitments 和 rate limits 管理高用量客户。

如果按纯 PAYG，小客户 churn 和价格竞争会限制估值；如果 Tavily 能把 search/extract/research 打包进企业 agent platform，并通过 Nebius 销售给大型客户，收入质量会显著提高。需要关注 Nebius 是否把 Tavily 作为 cloud AI infra 的附加服务，形成 cross-sell。

8竞争格局

8.1 与 Exa

Exa 是最直接竞品。Exa 定位为 Search API for AI agents，强调自研 AI search、Contents highlights、Deep Search、Agent、Monitors、Exa Connect；Tavily 定位为 secure API for real-time web access，强调 search/extract/crawl/research、安全、规模指标和 production readiness。

Tavily 相对 Exa 的优势:

  • 2M+ developers、300M+ monthly requests 等公开 usage 更强；
  • Nebius 提供基础设施和企业云资源；
  • 安全/PII/prompt injection narrative 更清晰；
  • 与 IBM、Databricks、LangChain、JetBrains 等生态集成。

Exa 相对 Tavily 的优势:

  • Benchmark/NVenture/Benchmark capital backing 和 $700M 估值；
  • Contents highlights/token efficiency 和 vertical search narrative；
  • Agent/Monitors/Exa Connect 进入数据 enrichment；
  • Cursor/Cognition/HubSpot/OpenRouter 等案例。

8.2 与 Perplexity

Perplexity 是 consumer AI search 入口，同时 API 化为 Search/Sonar/Agent API。Tavily 更专注 agent web access，不直接争 consumer 用户。对开发者来说，Perplexity 适合答案模型和引用输出，Tavily 适合可控 search/extract/crawl/research 工具。长期风险是 Perplexity API 用强品牌和模型输出质量抢开发者。

8.3 与 Google / Bing / Brave / SerpAPI

Google/Bing 拥有最大索引和默认搜索基础设施，但 API 输出并非专门为 autonomous agents 设计。Brave Search API 提供独立 web index，SerpAPI 提供搜索结果抓取。Tavily 的差异是 agent-specific output、安全、research workflow 和 developer docs。若 Google/Bing 将 agent-optimized search API 商业化，Tavily 的替代风险上升。

8.4 与 Firecrawl / Browserbase / Apify

Firecrawl 专注 crawl/extract，Browserbase 专注 browser automation，Apify 是 scraping/automation marketplace。Tavily 在 search + extract + research 层，可能与 Firecrawl overlap；但复杂页面交互和登录后网页仍可能需要 Browserbase/Apify。

8.5 与 Databricks / Dify / LangChain / GoSearch

Databricks Agent Bricks、Dify workflows、LangChain agents、GoSearch enterprise search 都可能调用 Tavily 作为外部 web tool。Tavily 的最佳位置是成为这些 builder 的默认 web tool，而不是自己做完整 app builder。Databricks MCP Marketplace 和 LangChain integrations 对其 distribution 很重要。

8.6 护城河与脆弱点

护城河:

  • 2M+ developer adoption；
  • 300M+ monthly requests 的生产规模；
  • 安全/PII/prompt injection controls；
  • Nebius infra 和潜在 cloud distribution；
  • search/extract/crawl/research 一体化；
  • 生态集成和开源起源。

脆弱点:

  • Search API switching cost 低；
  • Exa、Perplexity、Google、Brave、Bing、Firecrawl、Linkup 竞争；
  • Nebius 加入后独立品牌与战略优先级变化；
  • 内容授权和爬虫限制；
  • research endpoint 成本不可控；
  • 免费开发者规模不等于付费规模。

9团队与背景

Rotem Weiss 是创始人，TechCrunch 报道其数据科学背景，先做 GPT Researcher 开源项目，再转向企业级 Tavily。这个背景决定 Tavily 的产品基因: 不是传统搜索门户，而是研究/agent 工具。GPT Researcher 的 viral 带来早期开发者社区，随后 Tavily 将能力产品化为 API、SDK、docs、enterprise security。

加入 Nebius 后，团队需要从创业公司节奏转向基础设施公司节奏: 保持开发者体验、快速产品迭代和 community 同时，满足更大企业部署、SLA、全球低延迟和合规要求。若处理得好，Nebius 是放大器；若处理不好，可能降低独立产品敏捷性。

10主要风险

  1. 独立性变化: 加入 Nebius 后，产品路线可能服务 Nebius 云战略，而非所有开发者中立需求。
  2. Exa 竞争: Exa 获得 Benchmark/NVenture 支持，并有 $700M 估值和高质量客户案例。
  3. 模型厂商内置 search: OpenAI/Anthropic/Google/Perplexity 可把 web search 做成默认 tool。
  4. 价格压力: PAYG $0.008/credit 面临 search commoditization。
  5. 内容访问风险: Robots/paywalls/copyright/anti-scraping 会限制网页覆盖。
  6. 安全事故: prompt injection、PII leakage 或恶意网页导致 agent 错误，会严重影响企业信任。
  7. Usage gross margin: 300M+ monthly requests 规模下，crawl/extract/research 成本需要严格控制。
  8. 免费用户转化: 2M+ developers 中付费转化率不明。
  9. Enterprise sales cycle: 自定义 API、security review、SLAs 会拉长采购周期。
  10. 差异化可持续性: latency、uptime、安全和 citations 可能被竞品追平。
