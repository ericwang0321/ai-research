Zyphra 的独特性在于它不是只做模型,也不是只做云。官网把公司拆成两侧:Zyphra Research 负责开放模型和架构研究,Zyphra Cloud 把研究成果交付给 developers、enterprises 和 frontier AI hyperscalers。它的研究叙事是“open superintelligence”,商业叙事则是“sovereign control over AI”与“no lock-in”:客户可以拥有自己的模型、数据和部署,不被少数闭源平台或单一硬件生态锁死。

4业务模式与收费方式

4.1 当前商业化骨架:Zyphra Cloud

Zyphra Cloud 是公司从研究型模型公司转向商业平台的核心。官网描述它是“full-stack AI platform designed for advanced AI systems, with a focus on long horizon agents”。功能分为四层:

层级产品/能力潜在收费方式价值
Inferenceserverless inference、dedicated inference capacitytoken 用量、reserved capacity、SLA让客户运行 frontier open-weight models
ComputeGPU clusters、bare metal orchestration、dedicated capacityGPU-hour、裸金属租赁、长期容量合同对标 GPU cloud/AI factory
Agent EnvironmentsCPU environments、distributed training & RL、simulation environmentsenvironment-hour、training job、simulation usage支持 long-horizon agents 和 RL
Agent / MAIAmultiplayer general agent for teamsseat subscription、team workspace、enterprise license面向团队协作与工具编排

4.2 Zyphra Inference

Zyphra Inference 于 2026 年 5 月公开推出,定位是 AMD-first inference service,服务 long-context、agentic workloads。Zyphra 强调:

  • 面向大型 open models,不是只运行自家模型;
  • 优化长上下文和 long-horizon agent workloads;
  • 支持 serverless inference 和 dedicated inference capacity;
  • 通过 full-stack kernels、parallelism、communication overlap、quantization、speculative decoding 等系统工程提升 AMD MI355X/MI300X 性能。

潜在收费结构应包括按 token 的推理收入、专用 capacity 订阅、企业 SLA、裸金属/集群租赁、以及模型/agent 应用层 license。官网尚未给出公开 price card,因此不能写具体单价。

4.3 MAIA agent

MAIA 被描述为“general, open superagent for teams”,提供共享上下文、persistent memory、跨用户和工具的 coordinated execution,并支持 language/audio/vision 统一推理循环。商业化上,MAIA 更像企业协作型 agent:

  • 团队 seat subscription;
  • 企业 workspace / private deployment;
  • 工具和模型 orchestration 收费;
  • 与 Zyphra Cloud 的 inference 和 agent environments 打包;
  • 对特定行业流程做定制合同。

截至本报告日,MAIA 还没有公开客户价格、用户数或企业案例,所以应视为早期产品化方向而非已验证收入源。

4.4 Open model license 与企业 license

Zyphra 频繁采用 Apache 2.0 开源/开放权重策略,例如 ZONOS、ZAYA1-74B-Preview、ZAYA1-VL-8B 等公开页面提到 Apache 2.0。这有利于开发者传播,但对商业化提出要求:公司必须通过云推理、专用算力、企业部署、agent 产品和服务合同来变现,不能仅靠模型权重本身收费。

5.2 核心研究路线

Zyphra 官网把 research thesis 分成三支:

  1. Next-Gen Architectures:长上下文推理、长期记忆、continual learning,包括 hybrid associative memories、online vector quantized attention、compressed convolutional attention、adaptive depth 等;
  2. Multimodal World Models:语言、音频、视觉在共享 latent space 内训练,并扩展到 thought/EEG;
  3. Heterogeneous Compute:跨 AMD、AWS Inferentia/Trainium、NVIDIA 等硬件做 kernel、通信、并行和推理优化。

5.3 AMD-first 生态

Zyphra 最明显的战略变化是从“开源模型实验室”转向“AMD-first cloud/infrastructure company”。关键证据:

  • 2024-12 发布 AMD MI300X 上的 transformer/SSM 训练 kernels;
  • 2025-11 与 IBM Cloud/AMD 发布 ZAYA1,称为在 AMD hardware/software/networking stack 上训练的大型 MoE 模型;
  • 2026-05 发布 Zyphra Cloud,由 AMD Instinct MI355X GPU 支撑;
  • 2026-05 宣布 15MW AMD MI355X capacity;
  • 2026-05 发布 MI355X inference benchmark;
  • 2026-05/06 与 AWS Inferentia2 做 inference communication overlap 研究。

这条路线的投资意义在于:如果 AI 算力市场长期受 Nvidia 供给、价格和客户锁定影响,能够把模型和推理栈优化到 AMD/AWS 等异构硬件的公司会有战略价值。Zyphra 不是简单租 GPU,而是试图证明模型架构、kernel 和云服务可以共同适配非 Nvidia 硬件。

5.4 Long-context / agentic workloads

Zyphra 的研究主题长期围绕 memory 和 long context:

  • Mixture-of-PageRanks Retriever 声称可把有效上下文扩展到 billion-scale,并能在 CPU 上高效运行;
  • Tree Attention 设计用于多 GPU 长上下文 decoding,官方称在 1M sequence length 上相对 Ring Attention 有显著速度/通信优势;
  • Conversational agents with long-term memory 研究关注时间敏感和上下文敏感检索;
  • Zyphra Cloud 明确服务 long-horizon agentic workloads。

这与 MAIA 的产品化方向一致:agent 如果要跨天、跨工具、跨团队工作,长记忆和长上下文推理是关键基础设施。

Zyphra 最重要的运营指标不是单一模型 benchmark,而是:

7单位经济性 / 关键财务逻辑

Zyphra 的单位经济性要分三块看。

7.1 Inference cloud

收入来自 token 推理、专用容量、企业 SLA。成本来自 AMD GPU 折旧/租赁、电力、网络、存储、kernel 工程和调度。若 Zyphra 能证明 AMD MI355X 在 long-context MoE 推理上成本显著低于 Nvidia B200/H100,它就能在开源模型推理价格战中获得毛利空间。

关键变量:

  • MI355X 集群利用率;
  • prompt length 和 output length mix;
  • MoE 路由和 KV cache 优化;
  • speculative decoding 是否降低 cost/token;
  • dedicated capacity 是否提高可预测收入。

7.2 Bare metal / dedicated compute

如果客户租用专用 AMD capacity,收入更像云基础设施合同,毛利取决于硬件采购价格、电价、数据中心 PUE、融资成本和合约期限。15MW capacity 是重要信号,但没有客户利用率和合同期限,不能推算收入。

7.3 Agent / MAIA

MAIA 若按 seat 或 workspace 收费,毛利会比裸推理更像 SaaS,但底层 agent 运行会消耗大量 tokens、工具调用和 simulation environments。关键是能否把 agent 的自动化价值定价到 workflow outcome,而不是陷入 token 成本竞争。

7.4 开源模型的变现悖论

Apache 2.0/open weights 带来传播,但也让模型本身更难收 license fee。Zyphra 必须通过“最优运行这些模型的云”和“企业可控部署”赚钱。换言之,开源是 customer acquisition,云和企业合同才是 monetization。

MAIA 对标的是 OpenAI agent、Anthropic Claude workflows、Google Gemini Workspace、Microsoft Copilot、Perplexity enterprise agent、以及大量 enterprise agent startups。Zyphra 的差异是“open foundation models + sovereign deployment + long memory”,但 agent 产品要靠用户体验、工具集成和可靠性取胜,不能只靠模型技术。

8.4 护城河

  • 硬件适配能力:AMD/AWS Inferentia/NVIDIA 多硬件优化经验;
  • 模型架构研究:SSM、MoE、diffusion language models、long-context retrieval、BCI models 等多路线探索;
  • 开放生态:Apache 2.0 模型和数据集提升开发者信任;
  • 基础设施合作:IBM/AMD/TensorWave/AWS 等合作形成非 Nvidia 生态位;
  • 端到端 stack:从模型、数据、kernel、cloud、agent 到 enterprise deployment。

护城河风险是:每一层都有强敌,公司需要证明端到端整合真的比专注单点更有商业效率。

9团队与背景

Krithik Puthalath

创始人/CEO,公开资料显示其背景包括 quantum computing、machine learning、compiler 和 hardware-aware systems。Zyphra 的“模型架构 + silicon performance”路线与这一背景一致。

Beren Millidge

President/联合创始人之一,曾与 AI alignment、Conjecture、Apollo Research 等生态相关。Zyphra 的 open superintelligence 和 safety/alignment 叙事与其背景有关。

Tomas Figliolia / Danny Martinelli 等

公开资料将其列为联合创始或核心早期成员。公司还包括 COO/CPO/CBO/VP 等运营和产品角色,显示 2025-2026 年已从纯研究团队向商业组织扩展。

团队结构

Zyphra 的团队需要同时覆盖四类能力:

  • 模型研究:architectures、pretraining、post-training、multimodal;
  • 系统工程:kernels、ROCm、Neuron、distributed inference、bare metal;
  • 云产品:billing、capacity、SLA、enterprise support;
  • agent/product:MAIA、workflow、memory、tool orchestration。

这也是风险:对一家约数十人规模公司来说,战线很长。

10主要风险

  1. 战线过长:模型、云、agent、硬件优化、BCI、音频、视觉同时推进,资源分散。
  2. 云业务资本开支:15MW capacity 意味着硬件和电力投入重,需要高利用率才能支撑毛利。
  3. AMD 生态风险:若 AMD 软件栈、客户接受度或供应节奏不及预期,Zyphra 差异化会受损。
  4. 开源变现难:Apache 2.0 模型带来传播,但客户可以自部署或转向其他云。
  5. 模型能力风险:Zyphra 模型在特定 benchmark 上亮眼,但通用能力未证明能对抗 DeepSeek/Qwen/Mistral/Meta。
  6. 客户数据缺口:付费客户、ARR、云利用率、毛利率都没有公开。
  7. agent 产品可靠性:MAIA 要在真实团队场景中稳定执行,难度高于 demo。
  8. 安全与滥用:开放 TTS、voice cloning、agent 和 reasoning models 都有滥用风险。
  9. 基础设施竞争:CoreWeave、AWS、Azure、Google、Together、Fireworks 等资金和客户优势更强。
  10. 融资持续性:AI infra 估值对资本市场和 GPU/电力周期敏感。
