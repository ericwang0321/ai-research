Exa

Exa 是把 web search 重写给 AI Agent 用的搜索 API 公司。它不做 Perplexity 那样的消费者答案入口，也不做 Glean/GoSearch 那样的企业内部搜索，而是向 Cursor、Cognition、HubSpot、OpenRouter、chatbots 和 research agents 提供可编程网页检索能力。

1一句话结论

Exa 是把 web search 重写给 AI Agent 用的搜索 API 公司。它不做 Perplexity 那样的消费者答案入口，也不做 Glean/GoSearch 那样的企业内部搜索，而是向 Cursor、Cognition、HubSpot、OpenRouter、chatbots 和 research agents 提供可编程网页检索能力。

Exa 的价值在于让 Agent 找到、抽取和引用实时网页内容。投资上要看它能否在 Google/Bing/Brave Search API、Tavily、Firecrawl、Linkup、Perplexity API 和开源爬虫工具之间保持质量、速度、价格和开发者心智优势。

Exa 的核心论点是“AI 搜索行为会超过人类搜索行为”。传统搜索引擎为人类 SERP、广告、点击和网页浏览而设计；AI agents 需要的是低延迟、结构化、可引用、可控、token-efficient 的 web data。这个差异使 Exa 可以从 Google/Bing/Brave/SerpAPI 这些传统搜索 API 中切出新市场。

它与 Tavily 竞争最直接: 两者都卖 agent web access layer，均强调实时搜索、extract/crawl、citations、安全与企业支持。Exa 更强调自研 AI-native search index、highlights/token efficiency、FRAMES/Tip-of-Tongue/Seal0 benchmark、company/people/code verticals、Exa Connect 数据供应商；Tavily 更强调 production-grade retrieval stack、PII/prompt injection safeguards、2M+ developers、300M+ monthly requests 和 Nebius 资源支持。

官方 pricing 还披露 endpoint 级价格: Search $7/1k requests，Deep Search $12，Deep-Reasoning Search $15，Contents $1/1k pages，Monitors $15，Answer $5；超过 10 results 的额外 result 多数按 $1/1k requests；AI page summaries $1/1k pages。Agent pricing 包括 ACU $0.10/ACU、search tool call $0.005/search、email enrichment $0.02/email、phone enrichment $0.07/phone number，并按 Minimal/Low/Medium/High/X-high 固定 effort 收 $0.012/$0.025/$0.10/$0.50/$1.00/request。

4.3 Enterprise contract

Enterprise plan 强调 up to 1,000 results per search、more than 25 results、custom QPS、tailored moderation、custom index、SLAs/MSAs、1:1 onboarding/support。对于高频 AI app，custom index 和 QPS 可能是核心付费点；对于金融、法律、咨询、企业销售，数据授权、安全、zero data retention、SSO、audit 和 support 会驱动 enterprise ACV。

4.4 与 Perplexity / Tavily 的定价关系

Perplexity Search API 公开价约 $5/1K requests，Tavily PAYG 为 $0.008/credit，Exa Search 为 $7/1K requests。表面上 Exa 单价不最低，但它强调 token-efficient highlights、sub-180ms/1s configurable latency、code/company/people verticals 和 benchmark quality。投资判断要看客户是否愿意为质量、latency、结构化和数据供应商 access 付 premium。

5产品与技术

5.1 Search API

Exa Search API 为 agents 提供实时 web results、page snippets、highlights、domains filtering、contents options、structured outputs 和 citations。官网明确把 coding agents 列为核心 use case，称 Cursor uses Exa to get latest docs for more accurate code，并强调可搜索 GitHub repos、docs、Stack Overflow、changelogs 等。

Coding agent 场景非常重要，因为代码生成常常需要最新 API docs、package changes、GitHub issues 和 repo context。模型训练语料天然滞后，Search API 可以把最新文档和 repo 信息拉入 agent loop。

5.2 Contents API 与 highlights

Exa Contents 提供 full page contents 和 highlights。官网展示 highlights 可实现 90% token reduction，并提到 OpenRouter 每周最多 25T tokens 通过 highlights 被传给模型。这个产品点是 Exa 的核心差异: AI agent 不需要完整网页，而需要最相关、可引用、低 token 成本的 excerpt。若 highlights 质量高，可以显著降低下游 LLM 推理成本。

5.3 Deep Search / Agent / Monitors

Deep Search 把 search、crawl、reasoning、structured output、citations 组合成多步 research；Agent 则是异步 deep research、list building、enrichment run；Monitors 用 cadence 跟踪 web events，通过 webhooks 推送。三者把 Exa 从“search API”扩展到“agent research platform”。

这些产品适合:

  • PE/consulting: 公司筛选、市场地图、funding/news monitoring；
  • sales/recruiting: company/people enrichment；
  • coding agents: changelog/docs/repo search；
  • legal/finance research: 多源引用和监控；
  • voice/chat agents: 实时事实和可引用答案。

5.4 Exa Connect 与数据供应商

Exa Connect 提供 leading data providers access，配合 Agent/Deep Search 做 enrichment。它把 Exa 从开放 web 搜索扩展到授权数据、公司/人员数据和垂直数据源。这个方向对护城河重要，因为通用 web search API 很容易价格竞争；高质量数据供应商和 vertical datasets 更容易形成差异化。

5.5 Benchmark 与性能宣称

官网列出 FRAMES、Tip-of-Tongue、Seal0 等 retrieval benchmark，并声称 Exa 在 FRAMES、Tip-of-Tongue、Seal0 等上领先 Perplexity、Brave 等，Exa Instant under 180ms，且在 company search、people search、code verticals 领先。需要注意: 这些 benchmark 多为公司展示口径，投资分析中应作为产品 claims，而不是独立审计结论。真正的客户效果要看 Cursor/Cognition/HubSpot/OpenRouter 等生产集成是否持续扩大。

5.6 安全与控制

官网提到 zero data retention、SOC 2 Type II、SSO、team authentication/authorization。对 enterprise AI app 来说，搜索 API 可能接触客户 query、research target、公司名单和内部提示词。Exa 必须证明它不是会泄露客户研究意图的数据中间层。

商业化的核心不是“有多少开发者试用”，而是 API usage 是否进入高频生产链路。Cursor、Cognition、HubSpot、OpenRouter 这类客户若把 Exa 嵌入 agent loop，会带来持续 request volume；但 API 工具的 switching cost 通常低于数据平台，Exa 需要用质量、index、latency、verticals 和 enterprise support 提高留存。

7单位经济性 / 关键财务逻辑

Exa 的单位经济性由四个变量决定:

  1. Search infra cost per request: crawling、indexing、ranking、cache、storage、bandwidth 和 serving latency。低延迟 search API 若无足够 cache 和 index efficiency，毛利会被吞噬。
  2. Contents/highlights token savings: 如果 Exa 的 highlights 能显著降低客户 LLM token 成本，它可以用“为客户省推理成本”支撑较高 API 单价。
  3. Agent/Deep Search compute cost: 多步 research 和 structured extraction 需要更多 search/tool/model 调用，$0.012-$1.00/run 的 fixed effort 要避免高复杂任务亏损。
  4. Enterprise minimums: 大客户自定义 QPS、custom index、SLAs、security 能稳定收入，减少纯 PAYG 波动。

粗略看，Exa 比通用 SaaS 更重资产，因为要持续爬取和索引 web；但比模型公司轻，因为不需要训练 frontier LLM。长期毛利取决于自研 search index 的效率、cache 命中、数据供应商成本、LLM 用量和客户 willingness to pay。若 AI searches volume 真超过 human searches，规模优势会很强；如果模型厂商内置 search 成为默认，Exa 的流量入口会受压。

估值方面，$700M post-money 若没有 ARR 披露，很难用 revenue multiple 判断。更合理的监控指标是 monthly API requests、paid customers、gross retention、top customer concentration、enterprise ACV、search quality retention 和 infra gross margin。

8.1 与 Tavily

Tavily 是最直接竞品。两者都面向 AI agents 提供 real-time web access，且都在争取 LangChain、Databricks、IBM、Groq、MongoDB、Writer、Cursor 等生态。差异大致如下:

Exa 需要证明搜索质量和数据结构化明显优于 Tavily；Tavily 需要证明规模、Nebius infrastructure 和安全治理带来的 enterprise trust 更强。

8.2 与 Perplexity

Perplexity 既是 consumer answer engine，也通过 Search/Sonar/Agent API 进入开发者基础设施。Perplexity 的优势是品牌、consumer query data、publisher program 和 answer engine体验；Exa 的优势是更专注 developer API、token-efficient contents、vertical search 和 agent integration。若开发者只需要答案或 Sonar-style search model，Perplexity 更自然；若需要自定义搜索、contents、structured enrichment 和 agent run，Exa 更灵活。

8.3 与 Google / Bing / Brave / SerpAPI

传统 search API 的优势是索引规模、覆盖广、成熟度和稳定性。Exa 的差异是 AI-first output: 结构化、chunked、highlighted、citation-ready、agent-friendly。风险是 Google/Bing/Brave 一旦推出更适合 AI agents 的 API，Exa 的差异会被压缩。

8.4 与 Firecrawl / Linkup / Apify / Browserbase

Firecrawl 更偏 crawl/scrape/extract；Apify 是 web automation/scraping marketplace；Browserbase 是 browser automation infrastructure；Linkup 也是 AI search API。Exa 的边界是搜索+内容+research，而不是纯浏览器执行。很多 agent stack 可能同时使用 Browserbase 执行动作、Exa/Tavily 搜索、Firecrawl 抓取复杂页面。

8.5 与 Databricks / Snowflake / Glean / GoSearch

Databricks/Snowflake 是企业数据后端，Glean/GoSearch 是内部知识搜索，Exa 是外部 web data。企业研究 agent 通常需要三类数据: internal knowledge、structured enterprise data、external web。Exa 更可能与这些平台互补，而不是替代。

8.6 护城河与脆弱点

护城河:

  • AI-native search index 和 ranking；
  • token-efficient highlights 可能降低客户推理成本；
  • coding/company/people verticals；
  • data provider access via Exa Connect；
  • production customer integrations；
  • Benchmark/NVenture 等资本与 Nvidia 生态背书；
  • search quality evals 和 benchmarks。

脆弱点:

  • API switching cost 不高；
  • OpenAI/Anthropic/Google/Perplexity 可内置 web search；
  • web crawling 和内容授权政策变化；
  • 数据供应商成本和 compliance；
  • 若搜索质量领先无法持续，价格会被压低；
  • 大客户自建或转向云厂商 search。

9团队与背景

BI 2026 年报道提到 CEO Will Bryk 和 cofounder Jeff Wang，Exa 约 80 名员工，正在 San Francisco、Zurich、Singapore 招聘。Will Bryk 强调 hiring “rebellious” engineers、重视 first-principles、large-scale systems passion，并称候选人会在 San Francisco 与团队工作一到两天以评估文化和技术。报道还提到公司许多代码由 AI 写成，反映其自身也在采用 AI-native engineering workflow。

团队挑战主要是 infra scaling: search 公司不是纯 app，必须建设 crawler、index、ranking、serving、data pipeline、compliance 和 customer support。Singapore office 的重点被描述为 data pipelines 和 crawling infrastructure，这说明 Exa 正在从早期 API startup 进入全球化搜索基础设施建设阶段。

10主要风险

  1. 模型厂商内置 search: OpenAI、Anthropic、Google、Perplexity 可以把 web search 做成默认 tool，降低开发者采购第三方 API 的必要性。
  2. Tavily 竞争: Tavily 已有 2M+ developers、300M+ monthly requests、Nebius 背书，直接争夺 agent web access layer。
  3. 价格竞争: Search API 容易被按 request 比价，若质量差距不明显，价格会下行。
  4. 内容授权和爬虫限制: Robots、paywalls、法律诉讼和出版商政策会限制可访问数据。
  5. 数据供应商成本: Exa Connect 若依赖第三方数据，毛利受授权成本影响。
  6. API switching cost 低: 开发者可以抽象 search provider，尤其在 LangChain/LlamaIndex 等框架中切换。
  7. 搜索质量难衡量: 公司 benchmark 需要独立验证；客户生产质量更重要。
  8. 高估值前置: $700M 估值需要较快 ARR/API usage 增长支撑。
  9. 安全与隐私: 客户 query 可能包含商业敏感研究方向，ZDR 和审计必须可信。
  10. 基础设施扩张成本: 全球 crawling/indexing/serving 投入可能推高 burn。
