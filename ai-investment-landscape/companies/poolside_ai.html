Poolside AI

Poolside 已从 2024 年“尚未公开产品、但融资 $500M 的 coding model lab”，升级为全栈 AI coding 公司。到 2026 年，公司已公开发布 Laguna agentic coding 模型、Poolside Platform、政府安全部署方案和大规模 AI 基础设施计划。

1一句话结论

Poolside 已从 2024 年“尚未公开产品、但融资 $500M 的 coding model lab”，升级为全栈 AI coding 公司。到 2026 年，公司已公开发布 Laguna agentic coding 模型、Poolside Platform、政府安全部署方案和大规模 AI 基础设施计划。

公司 2024 年 10 月宣布 $500M 融资，媒体报道估值约 $3B；2025 年又出现 Nvidia 最高 $1B 战略投资、估值约 $12B，以及继续融资支持 2GW Project Horizon 数据中心的报道。投资关键是 Poolside 能否把安全边界内运行的 coding foundation models、Agent platform 和政府/企业部署转成真实收入。

Poolside 的核心判断是:软件开发是最先可被 AI 达到/超过人类能力的大规模经济活动之一,因为代码可以执行,执行结果能给模型稳定反馈。公司因此提出 RLCEF:模型在真实代码库和任务上尝试解决方案,通过代码执行反馈训练推理与编码能力。2024 官方募资文章称其模型在 130,000 个真实代码库 上探索数百万任务并接收执行反馈;2025 起又用 Model Factory 系列文章公开训练、数据、执行环境、推理评估和 post-training 流水线。

4业务模式与收费方式

Poolside 的收费逻辑与 GitHub Copilot / Cursor 的 $20-$40/seat SaaS 不同。它更像"模型权重 + 企业平台 + 安全部署 + 专业服务 + 算力/硬件生态"的组合。

4.1 政府 / 安全部署:不按 token 收费

Poolside 政府页明确写到,其面向政府机构的方案是 on-premises / air-gapped deployment,机构在自有硬件上运行推理,no per-token fees,从而把不可预测的 API 账单变成可规划的基础设施和订阅投入。

部署方式收费逻辑价值主张
Full stack inside boundary平台订阅 + 模型权重/部署 + 支持基础模型、推理、基础设施和安全整包交付
Modular模块化订阅/授权客户已有环境中嵌入 Poolside 模型和 agent runtime
Bring your own hardware模型/平台授权 + 部署服务客户用自有 GPU/服务器,降低数据外流
AWS / GovCloud / VPC云市场/企业合同通过 AWS 合同和安全体系采购
Dell turnkey hardware硬件合作 + Poolside 平台企业/政府更快部署 air-gapped AI

这意味着 Poolside 收入不是单纯 token GMV,而可能包括:

  • 年度模型 license / platform subscription。
  • 政府合同和系统集成合同。
  • Atlas inference engine / Poolside Platform 授权。
  • FDRE / solutions architect 专业服务。
  • 云市场采购和硬件合作渠道收入。

4.2 Poolside Platform 企业订阅

2026-05 官方发布 Poolside Platform,称其是可在客户安全边界内运行 AI agents 的生产级系统。平台提供:

  • 在 VS Code、Visual Studio、Zed、IntelliJ、terminal 等开发环境中运行 agent。
  • 连接 Slack、Jira、GitHub、Workday、Salesforce 等工具,通过 centrally managed MCP servers 管理。
  • 容器化执行环境、secret management、network policy control。
  • 记录每个 agent action、tool call、file edit、reasoning step 和 decision,形成 searchable trajectory。
  • 四种部署:bare metal、air-gapped turnkey hardware via Dell、AWS/Azure/GCP VPC。

公开材料没有给出标准 seat price。考虑其客户定位,更可能按企业合同、开发者席位、模型使用规模、部署方式和支持等级综合定价。

4.3 开发者 API / OpenRouter / Preview

2026-04 发布 Laguna XS.2 和 Laguna M.1 时,公司称两个模型在 API 和 OpenRouter 上限时免费使用。Laguna XS.2 为 Apache 2.0 open weights,有助于开发者试用和生态扩散;Laguna M.1 后续也开放权重。开发者 API 在早期更像获客和 benchmark 分发,真正收入重心仍在企业/政府部署。

4.4 基础设施战略

2025 年 Poolside 与 CoreWeave 宣布 Project Horizon 2GW AI campus 计划,媒体报道包括首期 250MW、可扩 500MW、CoreWeave 作为 anchor tenant / operating partner、40,000+ Nvidia GB300 GPU 集群等线索。若落地,Poolside 将从纯软件公司变成重资产 AI infrastructure + model lab。好处是获得长期算力成本控制;风险是资本开支、融资依赖和能源/建设执行压力极高。

5.2 Model Factory

Poolside 公开的 Model Factory 系列包括:

  • hidden engineering behind foundation model building。
  • data pipelines。
  • distributed training codebase Titan。
  • code execution environment。
  • inference and evaluation inside Model Factory。
  • post-training at scale。

这套体系说明 Poolside 的技术路线不是只训练代码补全模型,而是把数据、训练、执行、评估、post-training 全链路工业化。其特别强调代码执行反馈,因为代码任务可以通过测试、编译、运行和环境反馈形成自动监督。

5.3 Atlas inference engine

政府页称 Atlas 是 Poolside 的 inference engine,可提升模型性能 up to 30%。如果这项能力被客户采用,它会形成独立价值:同样硬件运行更多推理、在 air-gapped 环境中降低成本,也让 Poolside 不只是卖模型权重,而是卖可运行、可审计、可治理的完整系统。

5.4 安全与治理

Poolside 面向政府的功能包括:

  • Air-gapped and disconnected。
  • STIG-hardened OS。
  • ATO achieved。
  • Cleared resources。
  • IL5 deployable。
  • On-prem / GovCloud / edge。
  • Poolside Federal LLC、CAGE Code、NAICS Code、UEI。

这使公司与纯 IDE 插件明显区分。它瞄准的是"云 API 不能进入"的环境,包括国防、情报、关键基础设施、受监管企业。

7单位经济性 / 关键财务逻辑

7.1 与 Copilot/Cursor 的差异

GitHub Copilot、Cursor、Windsurf 等更多是开发者 seat SaaS / usage-based assistant;Poolside 更重:

  • 模型权重交付。
  • 私有部署。
  • 政府安全认证。
  • inference engine。
  • 解决方案工程。
  • 自有/合作算力。

这带来更高 ACV 潜力,也带来更长销售周期和更重交付成本。

7.2 政府 no-token-fee 经济学

客户视角:

  • 采购硬件/云资源 + Poolside license。
  • 推理无限或大规模内部使用,不怕 token 账单失控。
  • 数据不离开边界,合规价值高。

Poolside 视角:

  • 收入更像年度平台/模型 license,毛利理论上高于自营云 API。
  • 成本主要是研发、部署、支持和解决方案人员。
  • 如果客户使用自有硬件,Poolside 不承担全部推理变量成本。

7.3 基础设施资本开支

Project Horizon 使单位经济性复杂化。若 Poolside 自己承担大量数据中心资本开支,短期现金消耗会显著上升;但若 CoreWeave 作为 anchor tenant / operating partner 承担运营和部分负载,Poolside 可能以长期容量、能源和训练成本优势换取重资产风险。该项目需要重点跟踪:

  • 谁出资本开支。
  • Poolside 是否承诺最低采购。
  • 电力/天然气/水/土地审批进度。
  • GPU 采购和折旧。
  • 与 Nvidia 投资是否形成循环交易。

8.2 护城河

  • 创始人背景:Jason Warner 曾任 GitHub CTO,深刻理解开发者平台和企业软件工程;Eiso Kant 有软件创业背景。
  • 执行反馈数据:RLCEF 需要可运行代码环境和大规模任务反馈,比纯静态代码语料更难复制。
  • 安全部署能力:air-gapped、IL5、STIG、cleared personnel、Poolside Federal 等公共部门能力建立进入门槛。
  • 模型开放策略:Laguna XS.2 Apache 2.0 open weights 有利于开发者验证和生态。
  • 基础设施控制:与 AWS、Dell、CoreWeave、Nvidia 的绑定能缓解算力瓶颈。

8.3 脆弱点

  • 公开 benchmark 需要第三方验证,官网图表抓取异常说明外部可读性还有问题。
  • IDE / developer workflow 分发不如 Cursor 和 GitHub。
  • 政府销售周期长,收入确认慢。
  • 重资产算力策略可能让公司从软件高毛利变成资本密集型模型/基础设施公司。
  • $12B+ 估值需要非常高的收入增长或战略价值支撑。

9团队与背景

Jason Warner 曾任 GitHub CTO,也曾在 Redpoint 等投资/技术生态中活跃,对企业软件开发、代码协作、开源生态和开发者工具有深经验。Eiso Kant 是软件企业家,两人共同把 Poolside 的目标设为"让软件开发成为 AI 首个达到人类级/超人级能力的经济领域"。

公司采用分布式但定期集中工作的文化,美国和欧洲团队都很重要。2026 Laguna 文章提到约 60 人 Applied Research 组织参与模型发布,覆盖 infrastructure、architecture、data、pre-training、reinforcement learning。这说明团队结构已经从早期研究小组扩展为工业化模型团队。

10主要风险

  1. 估值风险:$12B+ 报道估值需要远超普通 devtool 的收入或战略价值支撑。
  2. 模型真实性风险:Laguna benchmark 需要独立验证,否则可能被 Cursor/Copilot/Claude Code 体验压制。
  3. 销售周期风险:政府/国防/on-prem 合同 ACV 高但周期长,收入兑现慢。
  4. 交付成本风险:FDRE、solutions architects、cleared personnel 会提高服务成本。
  5. 资本开支风险:2GW 数据中心计划可能吞噬现金,并暴露于电力、建设、GPU、融资市场变化。
  6. 竞争风险:GitHub/Microsoft、OpenAI、Anthropic、Google 可通过既有分发快速进入企业 coding agent。
  7. 开源替代风险:Qwen、DeepSeek、Devstral、Mistral 等 open models 可能以更低成本逼近能力。
  8. 政府合规风险:公共部门安全认证、出口管制、涉密项目都可能拖慢部署。
