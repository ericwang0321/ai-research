Clay

Clay 是面向销售、市场和 revenue operations 团队的 GTM 数据工作流平台。它把公司/联系人数据、第三方数据源、AI research agent、信号监测、CRM enrichment、广告受众同步和邮件触达整合到一个可编排的工作台里。

1一句话结论

Clay 是面向销售、市场和 revenue operations 团队的 GTM 数据工作流平台。它把公司/联系人数据、第三方数据源、AI research agent、信号监测、CRM enrichment、广告受众同步和邮件触达整合到一个可编排的工作台里。

公开资料显示，Clay 2017 年成立于 New York，2025-2026 年估值从约 $1.3B 上升到 $5B 员工流动性报道。投资关键是它能否从高增长的销售工具升级为企业 GTM 数据系统，而不是停留在 enrichment 和自动化工具。

Clay 的产品不应被归成“销售自动化 SaaS”这么泛。它更像 GTM 数据工作流层: 用户在表格式界面里导入 accounts/leads，调用 150+/200+ data providers 做 waterfall enrichment，用 Claygent 做网页和公司研究，跟踪 job changes/intent signals，把数据同步到 CRM、sequencer、LinkedIn/Meta/Google ads audiences，再用 AI 生成个性化 message 或 landing pages。它解决的是 GTM 团队最痛的“数据分散、供应商覆盖不完整、手工 research 费时、CRM 脏、outbound 个性化难规模化”。

在 AI 应用图谱里，Clay 与 Cursor/Lovable/Replit/Emergent 完全不同。后者让用户生成软件；Clay 让 revenue teams 更高效地找到、研究、触达、转化客户。它的成本结构也不同: 不只是 LLM token，而是第三方数据采购、email/phone enrichment、web scraping/research、AI agent、CRM/ads sync、support 和 enterprise implementation。

4业务模式与收费方式

4.1 Pricing: actions + data credits

Clay 2026 年官方 pricing 显示，它的核心计量有两类:

  • Actions: 工作流执行次数/操作量，用于自动化、表格处理、workflow steps。
  • Data credits: enrichment 数据调用额度，用于第三方数据、phone/email/company/person/job data 等。

4.2 Claygent 与 Waterfall 的经济意义

Clay 的两个关键产品概念:

  • Waterfall enrichment: 同一个字段用多个供应商依次查询，前一个 provider 没覆盖再调用下一个，以提高覆盖率并避免重复付费。对客户来说是 coverage/accuracy；对 Clay 来说是 provider routing 与数据成本优化。
  • Claygent: 有 web research 能力的 AI agent，可生成自定义数据点、研究公司/人、提取网页信息。对客户来说是替代手工 SDR/research；对 Clay 来说是 LLM/search/browser/data 成本。

这使 Clay 的单位经济性不同于普通 SaaS。它需要在用户愿意支付的 data credits/action 包与底层供应商/API/LLM 成本之间赚取毛利，同时让客户感觉比直接买 ZoomInfo/Apollo/People Data Labs/Clearbit/LinkedIn research 更有效。

4.3 Enterprise 收费

Enterprise 的价值包括:

  • bulk enrichment 与 bulk discounts；
  • security、governance、admin controls；
  • CRM/Salesforce 深度集成；
  • custom workflows、data governance、retention；
  • solutions partner / implementation support；
  • 高用量 actions/data credits 合同；
  • procurement、DPA、SOC2/trust center。

Clay 的客户类型决定 enterprise 非常重要。GTM data 往往进入 CRM、ads audiences、sales sequences 和 customer systems，涉及 PII、email/phone、GDPR/CCPA、CAN-SPAM、deliverability 和数据来源合规。企业愿意为可治理的数据工作流支付更高价格。

4.4 成本结构

Clay 的成本项:

  1. 第三方数据 provider 成本: email、phone、company、job、technographic、firmographic、intent。
  2. LLM/AI agent 成本: Claygent research、AI formatting、message generation、Sculptor natural language workflow creation。
  3. Search/browser/scraping 成本: web research、signals、job changes、site monitoring。
  4. CRM/ads integration 成本: Salesforce/HubSpot、LinkedIn/Meta/Google audiences、reverse ETL。
  5. Deliverability/compliance/support: outbound use cases 带来合规和 reputational risk。
  6. Human success/solutions: GTM teams 往往需要 workflow design、templates、best practices 和 agency ecosystem。

5.2 工作流例子

一个典型 Clay workflow:

  1. 从 CRM、CSV、LinkedIn list 或 website visitor 导入 accounts/leads。
  2. 用 Waterfall 依次查公司域名、员工数、职位、邮箱、手机号、技术栈。
  3. 用 Claygent 读取公司网站、招聘页、新闻、podcast、SEC/press release，生成 custom account research。
  4. 用 AI formatting 把原始字段整理成 outbound personalization。
  5. 根据 job change、funding、hiring、intent signal 触发 task。
  6. 同步到 Salesforce/HubSpot 或 sequencer。
  7. 把 audience 推到 LinkedIn/Meta/Google 做广告。

Clay 的壁垒在于 workflow flexibility。ZoomInfo/Apollo 更像数据库 + sequencing；Clay 更像 GTM spreadsheet + programmable data/AI workflow。

5.4 AI 化方向

Clay 的 AI 化不只是“帮你写邮件”。更重要的是:

  • 用 Claygent 自动做网页研究和定制字段；
  • 用 Sculptor 让用户用自然语言创建 GTM workflows；
  • 用 AI formatting 清理和标准化数据；
  • 用 signals 自动触发下一步行动；
  • 用 MCP 把 Clay 变成 AI agents 可调用的 GTM data/action layer。

如果 AI agents 未来能代表用户执行销售/市场任务，Clay 有机会成为这些 agents 的数据和行动底座。

7单位经济性 / 关键财务逻辑

7.1 收入端

Clay 收入可以拆成:

  • subscription plan base；
  • actions volume；
  • data credits volume；
  • phone/email/company/job enrichment；
  • enterprise bulk data；
  • Salesforce/CRM/ads audiences integrations；
  • solutions/implementation/support；
  • potential marketplace margin from data providers。

7.2 成本端

Clay 成本并非传统软件 90% gross margin:

  • 第三方数据 provider 成本可能随 data credits 增长；
  • phone number enrichment 通常比 email/company data 更贵；
  • Claygent web research 消耗 LLM/search/browser resources；
  • 数据质量失败、bounce、deliverability 会带来 support；
  • enterprise onboarding 和 workflow design 需要 human success。

7.3 毛利关键

Clay 的毛利取决于:

  1. Waterfall routing 是否减少无效 provider calls；
  2. data credits 定价是否覆盖底层成本；
  3. 客户是否愿意为 workflow/AI layer 支付软件溢价，而不是把 Clay 当数据 pass-through；
  4. enterprise bulk discounts 是否牺牲毛利换规模；
  5. BYO API key 占比。如果客户带自己的数据 provider key，Clay 少赚 data margin，但软件粘性和低成本可能更强。

7.4 ROI 逻辑

Clay 的客户 ROI 通常来自:

  • enrichment coverage 提升；
  • SDR/research 人工时间下降；
  • outbound personalization 提高回复率；
  • CRM 数据质量改善；
  • faster experiment cycles；
  • ads audiences 更精准；
  • pipeline contribution。

公开 case 中 Intercom +140% outbound-sourced pipeline、OpenAI 2x enrichment coverage、Rippling 2x cold email performance，都是典型 ROI 叙事。但这些是客户案例，尽调时需要看样本、基准、归因和是否可重复。

8.2 Clay 的价值链位置

Clay 位于 data providers 之上、CRM/engagement/ads systems 之前:

  • 输入: CRM、CSV、website visitors、LinkedIn lists、product data。
  • Enrichment: providers、Waterfall、Claygent、signals。
  • Transform: AI formatting、filters、scoring、segmentation。
  • Action: Sequencer、Salesforce/HubSpot、ads audiences、webhooks、MCP。

这使 Clay 既可能与 ZoomInfo/Apollo 竞争，又可能把它们作为数据 provider 使用。Clay 的长期价值在于成为 GTM data orchestration layer，而不是单一数据源。

8.3 护城河

  1. Workflow flexibility: GTM ops 可以像 spreadsheet 一样快速搭建复杂流程。
  2. Data provider marketplace: 多源 waterfall 使客户不被单一数据库覆盖率限制。
  3. AI research layer: Claygent 可生成自定义、难结构化的 account insights。
  4. Community/templates: GTM engineers、experts、templates 形成生态。
  5. Customer logos: OpenAI/Anthropic/Rippling/Intercom 等高增长 AI/SaaS 客户提高品牌可信度。
  6. Action layer expansion: Sequencer、Audiences、Salesforce、MCP 让 Clay 从 enrichment 进入 activation。

8.4 可被攻击点

  • Salesforce/HubSpot/Apollo/ZoomInfo 可把 AI enrichment/workflows 内置。
  • Data providers 可涨价或限制 resale/routing。
  • 客户自建数据栈可替代高用量 Clay workflows。
  • outbound automation 可能受 spam、deliverability 和监管限制。
  • 如果 AI SDR agents 端到端完成 prospecting，Clay 需要成为底层 data layer 而非被绕过。

9团队与背景

Clay 由 Kareem Amin 与 Varun Anand 于 2017 年创立。First Round Review 曾写过 Clay 寻找 product-market fit 的路径，说明公司并非一开始就直接命中现在的 AI GTM 形态，而是经过多年迭代才找到“数据 + workflow + GTM teams”的强 PMF。Kareem Amin 的公开访谈常强调 Clay 的目标不是让 GTM 变成机械化 spam，而是让团队更精准、更个性化、更有创造性地触达客户。

团队文化上，Clay 官网强调 creative、close-knit、growth mindset，并在 New York 建立强品牌与社区。Clay 的 GTM engineer 社群、模板、University、experts marketplace 和 Sculpt conference 都是公司把产品、教育和 community 合在一起的方式。

10主要风险

  1. 数据合规风险: email/phone/person data 涉及 GDPR、CCPA、CAN-SPAM、data provenance。
  2. Provider dependency: 数据供应商涨价、限制 resale 或 coverage 下降，会影响毛利和质量。
  3. Outbound saturation: AI 个性化 outbound 泛滥可能降低回复率并伤害 deliverability。
  4. 竞争风险: ZoomInfo/Apollo/Salesforce/HubSpot/6sense 可内置类似 AI workflows。
  5. 客户自建风险: 大企业可能用 Snowflake/Databricks/dbt/LLM 自建 GTM data workflows。
  6. 估值风险: $3.1B-$5B 需要强 ARR、gross margin 和 enterprise retention 支撑。
  7. ROI 归因风险: pipeline uplift 受市场、message、sales team 执行影响，难完全归因 Clay。
  8. AI agent 替代风险: 端到端 AI SDR 若直接集成数据源和 CRM，Clay 需要守住 data/workflow layer。
