阅读方法

AI 产业的价值并不会只留在模型公司。模型决定能力上限，但应用掌握用户入口，数据决定回答是否可靠，工作流决定产品是否真正进入业务，基础设施则承接训练、推理、部署和治理。长期价值更可能流向能够控制关键入口、沉淀专有数据、形成高切换成本，并把技术优势转化为稳定收入和现金流的公司。

这份导读按六个赛道组织，每家公司用一段话回答五个问题：它做什么，和同类相比有什么不同，向谁收费，为什么客户愿意持续付费，以及现阶段是否值得进入下一轮尽调。判断尽量使用公开可回源的收入、客户、用户、用量、融资和估值信息；没有披露的数据不硬估，也不因为缺失就按零处理。

“优先深研”不是给公司打分，更不是按估值从低到高挑公司。本篇先看两件更接近长期竞争力的事情：公司是否控制高质量、可合法使用的数据，并让真实使用持续产生更好的训练或评估信号；公司是否针对一个高价值领域改造模型、检索、工具调用和质量评估，而不只是把通用 API 接到一个界面上。收入、估值、融资条款和退出路径仍然重要，但它们是判断技术与商业闭环之后的第二层约束。

为什么先看数据飞轮与模型适配

高质量数据不等于“接入了很多数据”

基础模型的通用能力会扩散，推理价格也会下降，因此应用公司的长期价值不能只来自调用了哪一个模型。真正稀缺的是别人无法合法、低成本复制的数据，以及把这些数据转化为结果标签的能力。这里需要区分三类资产：公司拥有或长期控制的专有数据，例如独家授权内容、实验样本和自建索引；客户授权平台处理的私域数据，例如病历、合同和内部知识；任何人都能购买、抓取或接入的第三方数据。第二类数据非常有价值，但通常归客户所有，也可能受隐私、保密和跨租户限制，不能直接当成公司的训练资产。

数据飞轮只有在“使用产生反馈，反馈改善产品，产品改善又带来更多高质量使用”时才成立。一次医生修改、一笔理赔结果、一项实验验证、一段客户服务是否真正解决问题，都可能成为比原始文本更有价值的结果标签。反过来，如果公司只是保存更多文档，却没有权利复用、没有稳定标签，也没有把反馈进入训练、路由或评估体系，数据量再大也只是存储规模，不是护城河。

垂直适配不等于换一个提示词

直接调用通用模型可以很快做出产品，但同类公司也能用相似成本复制，基础模型厂商还可能把功能直接打包。真正的垂直适配需要把领域知识和业务结果写进整套系统：选择或训练专用模型，建立领域检索与重排，限制工具权限，为特定任务设计评估集，并在延迟、成本、合规和人工升级之间持续优化。判断标准不是“用了自研模型”这句话本身，而是适配后的系统是否在生产任务上稳定优于通用 API，是否减少人工介入或单位成本，以及优势是否会随着使用积累而扩大。

因此，本篇把“高质量数据 × 可观察结果 × 领域适配 × 工作流分发”视为一个完整闭环。四项不必都由公司完全拥有，但必须说明数据权利、反馈机制和价值归属。以下公司不是按精确名次排列，也不受统一估值上限约束；它们只是目前最值得先验证这一闭环的研究对象。

优先深研：数据和适配已经进入生产闭环

Abridge：临床对话不是终点，结果标签才是核心

Abridge 已从环境式病历延伸到护理文档、收入周期和临床决策支持。公司在 2026 年 6 月披露已进入 300 多个医疗系统、每年支持超过 1 亿次临床对话，这意味着它能在真实电子病历流程中观察医生如何修改记录、编码是否被接受、文档是否完整以及后续工作是否顺利完成。规模来源 与 Ambience Healthcare、Nabla 和 Suki 相比，Abridge 的公开部署规模和工作流宽度更强；Ambience Healthcare 在专科编码与收入完整性上的适配更深，但公开系统覆盖仍较少。

这里最重要的尽调问题不是“有多少对话”，而是病历和语音归谁、哪些去标识化反馈可用于跨客户评估、医生修改能否转化为可复用标签，以及模型改善能否减少拒付和人工返工。如果这些权利和效果成立，Abridge 控制的是临床事实进入医疗系统的入口，而不只是一个记录工具，因此进入第一组深研。

Decagon AI：客服 Agent 中最直接的模型适配证据

Decagon AI 是当前 Landscape 中最符合“不是简单套 API”这一标准的客服公司。公司在 2026 年 3 月披露，超过 80% 的模型流量运行在自己训练的模型上，并说明其为语音端点检测、工作流执行和幻觉识别等任务分别训练专用模型；这些模型服务于数百万次真实客户交互。技术来源 这比泛泛宣称“自研模型”更有意义，因为它把模型选择、后训练、评估、低延迟和客户结果连接在同一生产系统里。

与 Sierra 相比，Decagon AI 的公开收入规模较小，但自训练模型占比和产品化反馈回路更清楚；Sierra 的结果计费和大客户分发更成熟，同时明确不使用客户数据训练模型，因此其壁垒更依赖工作流、路由和评估，而不是跨客户原始数据。Writer 与 Cohere 覆盖更广的企业 AI 场景，却没有同样聚焦的客服结果标签。应优先核验 Decagon AI 的训练数据授权、每个已解决问题的模型后毛利、人工升级率和客户续约，而不是只看自动解决率。

AlphaSense：内容权利与研究工作流共同构成壁垒

AlphaSense 的优势不是“金融版聊天机器人”，而是把券商研究、电话会、专家访谈、公司文件和结构化金融数据放进一套可追溯的研究流程。公司在 2026 年 6 月披露内容库超过 5 亿份商业文档、服务 7,000 多家企业，Q1 2026 ARR 超过 6 亿美元。公司来源 其中客户上传的内部文件仍属于客户，不应被误写成 AlphaSense 可跨客户训练的私域数据；真正可防守的资产是长期授权的高价值内容、Tegus 专家访谈、领域检索经验和研究人员的查询与引用反馈。

与 Glean Technologies 相比，AlphaSense 对外部专业内容的控制更强；与 Hebbia 和 Rogo 相比，它的内容规模、客户基数和商业化证据更成熟。需要验证的不是文档数量，而是独家内容占比、授权成本、生成式功能带来的留存和增购，以及客户最终是否仍需回到 Bloomberg、FactSet 或 Capital IQ 完成关键判断。

Enveda Biosciences：最接近物理世界闭环的数据飞轮

Enveda Biosciences 不是用公开论文训练模型后向药企卖软件，而是收集植物与天然样本，用质谱产生化学数据，通过 PRISM 模型识别候选分子，再用自动化实验和生物筛选验证，最终把结果送入自有药物管线。PRISM 使用超过 10 亿条小分子质谱训练；公司公开管线已有 3 个临床资产覆盖 4 个适应症、6 个 IND-enabling 资产和 17 个开发候选。模型来源 管线来源

与 Isomorphic Labs 相比，Enveda Biosciences 的结构预测平台声量较小，但天然样本、质谱和湿实验数据更独特；Bioptimus 与 EvolutionaryScale 更偏基础模型和平台授权，尚未形成同样清楚的自有资产价值捕获。它的闭环最完整，风险也最硬：AI 只能提高发现效率，不能替代人体疗效。第一组深研应把 ENV-294 的 Phase 2a 结果、样本与质谱数据的独占程度，以及每轮实验是否真实提高命中率放在估值之前。

Kling AI / 可灵AI：模型、创作反馈与分发在同一生态内

Kling AI / 可灵AI 同时具备自研视频模型、创作者产品、API 和快手内容生态。快手披露可灵在 2026 年第一季度收入超过 RMB650M，2026 年 3 月年化收入运行率约 US$500M；这比只披露生成次数更能说明模型能力已经转化为付费。业绩来源 与 Runway、AIsphere / PixVerse、Luma AI 和 Pika 相比，可灵的模型、收入和分发证据最完整；Runway 更深入专业影视流程，AIsphere / PixVerse 的估值弹性更大，但都没有同等规模的集团内容入口。

可灵的难点也正来自集团生态。快手平台上的视频、用户偏好和广告转化数据是否能够由拆分后的主体持续、合规、低成本使用，训练数据、IP、流量和算力如何通过关联交易定价，都会决定所谓数据飞轮最终属于可灵还是快手。它值得优先研究，但必须以独立财务和数据权利为前提，不能把集团资源无条件算进创业公司护城河。

OpenEvidence：用授权医学内容占据医生决策入口

OpenEvidence 的核心数据不是普通网页，而是与 NEJM、JAMA、NCCN、Cochrane 和 Wiley 等机构签署的内容协议，再叠加已验证医生的高意图临床查询。公司称累计支持超过 2 亿次 AI 临床咨询。公司来源 与通用模型相比，这些授权内容和引用体系更难复制；与 Abridge 相比，它不控制病历生成和收入周期，而是控制医生查证医学证据的决策入口，两者的数据和商业模式并不相同。

OpenEvidence 的飞轮需要更谨慎验证：医生查询能揭示真实临床需求，但不自动等于正确诊疗结果；广告和赞助内容又可能与医学中立发生冲突。应重点核验付费方结构、内容授权成本、医生复访、引用后的行为反馈，以及模型在独立临床评估中是否持续优于通用模型。它进入第一组，不是因为流量大，而是因为高质量内容权利和专业用户入口可能共同形成新的医学知识分发层。

第二组观察：数据资产很强，但飞轮证据尚未闭合

Ambience Healthcare 是最接近第一组的医疗公司。它披露企业部署利用率超过 80%、覆盖 200 多个专科，并把专科文档、编码和收入完整性写进模型与工作流。产品来源 相比 Abridge，它的专科适配证据更直接，但客户规模、年处理量和收入尚未同样清楚。若能证明编码改善、拒付下降和模型学习之间的因果关系，它可以上调。

Harvey 已进入大量律所与企业法务流程，ElevenLabs 也拥有自研语音模型、声音授权和广泛 API 使用，两者都不是简单包装产品。但法律文件受客户保密与律师特权约束，语音数据又受版权和人格权限制，数据量不能直接等同于可复用训练资产。Harvey 需要证明工作流反馈和专用评估能形成跨客户产品改进；ElevenLabs 则需要证明授权声音、企业语音 Agent 和真实交互反馈持续改善模型，而不是只靠模型规模与分发领先。

Character.AI 拥有角色创作、长对话和用户偏好数据，在消费应用中具备少见的反馈密度；但核心创始人与部分研究团队回到 Google 后，剩余团队能否继续把这些数据转化为模型提升并不清楚，未成年人安全和诉讼也会限制数据使用。Corgi 若能把报价、承保、续保、理赔和最终损失率连成标签，可能形成保险领域非常强的数据闭环；目前公开证据主要是收入与客户增长，综合成本率、赔付率和模型带来的承保提升仍未披露。

Sierra 的商业化、结果计费和大型客户分发明显强于多数客服 Agent，但公司强调客户数据不用于训练模型。这个承诺有利于隐私与企业信任，却意味着其飞轮更可能来自匿名化评估、工作流模板、模型路由和工程经验，而不是汇集客户原始数据。与 Decagon AI 相比，Sierra 更像商业领先者，Decagon AI 则给出了更直接的专用模型证据，两者应使用不同的尽调问题。

为什么增长很快仍可能暂不优先

Lovable、Genspark、Manus 和 Granola 都可能建立很好的产品入口和用户习惯，但目前更依赖第三方基础模型。它们可以用项目、操作轨迹和用户修改改善编排、界面与评估，却不一定能让底层能力形成独占差异。研究重点应是工作流留存、切换成本和模型降价后能保留多少毛利，而不能因为收入增长快就自动推断出数据护城河。

Clay 把多源数据和 GTM 执行组织得很好，Exa 也拥有自己的网页索引，但两类资产与高质量私域数据仍有区别。Clay 的大量数据来自第三方供应商，优势更像工作流和供应商编排；Exa 控制公共互联网检索基础设施，技术壁垒真实，却不是一个垂直行业的私域结果闭环。它们值得作为数据与基础设施标的研究，只是不应和医疗、保险或药物发现的闭环用同一套理由入选。

Fireworks AI、Baseten 和 Together AI 能从推理流量中积累性能数据并优化成本，但客户可以迁移模型与算力，护城河更依赖软件体验、利用率、合同和单位经济性。另一方面，OpenAI、Anthropic、Moonshot AI 和 DeepSeek 确实拥有模型、用户和开发者反馈，但它们参加的是通用能力、算力和分发的资本竞赛，不是本节寻找的垂直数据复利。它们没有因为估值被排除，只是需要另一套模型领先持续性和资本回报框架。

通用 AI

通用 AI 赛道同时覆盖前沿基础模型、高效端侧模型、通用 Agent 与依托第三方模型形成的生产力应用。模型公司的价值主要来自能力领先、开发者生态、企业分发与算力效率；应用公司的价值则取决于能否占据高频入口，把模型能力沉淀为稳定工作流、用户数据和可重复付费。

投资判断不应只看模型榜单或产品热度，而要区分技术领先能否转化为收入、毛利和留存。重点观察 API 与订阅的付费质量、企业合同及部署深度、推理和训练成本、核心团队稳定性与平台依赖；对尚无产品或收入的研究实验室，则应按高方差技术期权而非成熟软件公司审视。

Anthropic

该公司研发前沿大模型，并以 Claude、代码 Agent、API 和企业服务切入高价值知识工作。独特点是安全品牌、长上下文、代码能力及云渠道协同；收入来自个人与团队订阅、企业席位和 API 用量。其商业价值在于成为企业可信模型与开发工具入口，但高估值、算力成本和云平台依赖仍需验证。它属于通用模型与分发的资本密集型框架，而非本篇优先寻找的垂直数据闭环，现阶段更适合作为行业标杆。

DeepSeek / 深度求索

该公司研发推理与通用大模型，以开源权重、低成本训练推理和低价 API 形成行业影响力，独特点是工程效率及对模型价格体系的冲击。收入方向主要是 API 用量与企业服务，商业价值还包括国产模型生态和开发者心智。其客户付费、毛利、治理及跨境合规仍缺少充分披露。它值得按模型效率和开放生态继续跟踪，但不属于垂直私域数据驱动的优先样本，商业化与财务可验证性也不足。

Genspark

该公司把搜索、研究、文档、演示、多媒体生成及业务操作整合为 AI 工作空间，独特点是跨工具完成可交付成果，而非只提供答案。收入来自个人订阅、团队席位、企业合同和任务额度，商业价值是成为中小团队的通用生产界面。其壁垒受制于第三方模型、同类平台内置功能和高成本任务的单位经济性。暂不优先：产品覆盖广，但留存、毛利与企业扩张尚待证明。

Granola

该公司提供会议笔记与团队记忆工具，结合用户主动记录、后台转写、会后总结和跨会议检索；不需要机器人账号加入会议、可积累团队上下文并连接外部工具，是其主要特点。收入来自个人免费增值、团队与企业按席位订阅。商业价值在于把会议沉淀为可供外部 Agent 调用的组织数据层，但会议平台原生功能可能压缩空间。暂不优先：会议记录属于高频场景，但独立上下文层的长期护城河仍需验证。

Liquid AI

该公司研发非 Transformer 的高效基础模型，面向端侧、边缘和私有部署，独特点是更低内存占用及与芯片、汽车和设备生态的适配。收入模式以商业授权、OEM、私有化部署和企业合同为主，而非纯按 token 售卖。其商业价值是打开云端巨头较弱的本地推理市场，但架构优势可能被主流模型优化追平，销售兑现周期也较长。暂不优先：路线差异鲜明，收入规模化仍待验证。

Manus / Butterfly Effect

该公司提供能调用多模型、云端电脑、浏览器和外部工具的通用 Agent，可从自然语言任务生成研究、网页、演示及跨系统操作。独特点是工作流广度与交付速度；收入来自任务额度、团队计划、连接器和 API。商业价值在于把聊天升级为可执行自动化，但底层模型依赖、任务可靠性和毛利仍有不确定性。暂不优先：Meta 收购已于 2026 年 4 月被中国监管部门叫停并被要求撤回，当前独立经营状况及后续交易安排仍需核验。

如何处理其余公司

“暂不优先”不等于公司差，而是目前还不能证明竞争优势会随使用复利。最常见的断点是：客户数据不能跨租户使用，反馈没有形成清楚的结果标签，产品仍依赖通用模型且容易被复制，收入随实施人力或算力同步增长，或者公司并不控制关键内容、分发和交易入口。估值过高、交易状态变化、医疗责任、版权和跨境监管仍会影响最终回报，但这些是闭环成立之后才应讨论的交易与风险条件。

这些判断是动态的研究优先级。若公司后来能够证明数据授权、生产反馈、专用模型效果和单位经济性之间存在可重复关系，即使当前收入较小，也可以上调；反过来，第一组公司如果不能说明数据归属、跨客户学习边界、相对通用模型的生产提升或客户续约，也应降级。公开叙事只能决定先看谁，不能替代数据室和客户访谈。

结论

AI 时代真正有复利能力的应用公司，往往不是功能最多的公司，而是能把高质量数据、明确结果和领域模型适配连接起来的公司。最有价值的数据也不是原始文本本身，而是医生如何修改、客户问题是否解决、理赔最终是否发生、实验结果是否验证候选分子这些带结果的反馈。模型越通用，垂直公司越需要用这种反馈建立自己的评估和后训练体系。

下一轮尽调应按这个顺序展开：先确认数据的所有权、授权范围和保存期限，再看多少生产流量真正运行在自有或深度适配的模型上；随后用同一任务比较通用 API 与公司系统的准确率、人工升级率、延迟和单次成功成本，最后才把留存、毛利、客户集中度、估值和股权条款放进回报测算。只有当技术改善能够转化为客户结果，并让下一次交付更好或更便宜，数据飞轮才是投资事实，而不是一句宣传语。

本页基于公开资料整理。私营公司数据可能来自公司披露、权威媒体报道或明确标注的第三方估算；实际投资仍需核验财务报表、客户合同、股权条款和监管状态。
